1. Огляд

  Більшість часу Tor трактує визначені користувачем імена вузлів прозоро: коли користувач з’єднується з www.torproject.org, Tor обирає вихідний вузол та використовує цей вузол для з’єднання до "www.torproject.org". ОДнак деякі імена вузлів можуть бути використані для зміни поведінки Tor по замовчанню та правила побудови з’єднань.

  Ці імена можуть бути передані до Tor як частина адреси запиту SOCKS4a або
  SOCKS5. Якщо застосування з’єднано до Tor лише з використанням методу IP
  (такого, як SOCKS4, TransPort, або NATDPort), ці імена можуть бути підмінені для окремих IP адрес з використуванням опції конфігурації MapAddress, або команду управління MAPADDRESS.

2. .exit

  СИНТАКСИС:  [hostname].[name-or-digest].exit
              [name-or-digest].exit

  Ім’я вузла є дійсним іменем вузла; [name-or-digest] є або нікнейм вузла   Tor, або закодований в шістнядцітиричній формі дайджест публічного ключа того вузла.

  Коли Tor бачить адресу в цьому форматі, він використовує вказане ім’я вузла як вихідний вуздол. Якщо компонента "hostname" не надається, Tor по замовчанню до опублікованої адреси вихідного вузла.

  Є допустиме спробувати розрішити ім’я вузла, та, фактично, в разі успіху Tor
  буде кешувати внутрішній mapaddress в формі "www.google.com.foo.exit=64.233.161.99.foo.exit" для прискорення подальших оглядів.

  Нотація .exit відключена по замовчанню, починаючи з Tor 0.2.2.1-alpha, через потенційні атаки рівня застосування.

  ПРИКЛАДИ:
     www.example.com.exampletornode.exit

        З'єднатися з www.example.com з вузла на ім’я "exampletornode".

     exampletornode.exit

        З'єднатися з опублікованою IP "exampletornode" з використанням     "exampletornode" як виходу.

3. .onion

  СИНТАКСИС:  [digest].onion

  Дайджест - це перші шістнадцять біт хешу SHA1 ключа ідентичності для прихованого сервісу, закодовані в base32.

  Коли Tor бачить адресу в цьому форматі, він намагається виконати пошук та з’єднатися до вказаного прихованого сервісу. Дивіться rend-spec.txt для повних деталей.

4. .noconnect

  СИНТАКСИС:  [string].noconnect

  Коли Tor бачить адресу в цьому форматі, він безпосередньо зачиняє з’єднання без приєднання його до жодного з’єднання. Це корисно для контролерів, що бажають протестувати, чи дане застосування прискіпливо використовує той же екземпляр Tor, що вони контролюють.

  Ця функція була додана в Tor 0.1.2.4-alpha, та видалена в Tor
  0.2.2.1-alpha через страхи, що це надасть додатковий шлях для детекції користувачів Tor через трюки на рівні застосування.
