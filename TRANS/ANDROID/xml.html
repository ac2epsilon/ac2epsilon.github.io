Extensible Markup Language (XML) є набір правил для кодування документів в формі, доступоному для читання машиною. XML є популярним форматом для поширення даних в інтернеті. Веб сайти, що часто оновлюють свій вміст, такі, як сайти новин або блоги, часто провадять XML фіди (потоки), так що завнішні програми можуть слідувати за змінами вмісту. Завантаження та розбір даних XML є загальним завданням для під’єднаних до мережі застосувань. Цей урок пояснює, як розбирати документи XML та використовувати їх дані.

Вибір синтаксичного анализатора (парсера)

Ми рекомендуємо XmlPullParser, що є ефективним та керованим шляхом для розбору XML під Android. Історично Android має дві реалізації цього інтерфейсу:

  • KXmlParser через XmlPullParserFactory.newPullParser().
  • ExpatPullParser, через Xml.newPullParser().

Кожний вибір є гарним. Приклад в цьому розділі використоує ExpatPullParser, через Xml.newPullParser().

Аналіз фіду

Перший крок в розборі фіду є вирішити, які поля нас цікавлять. Парсер виділить дані з ціх полів та ігнорує інші and ignores the rest.

Ось витяг з фіду, що ми будемо розбирати в прикладі застосування. Кожне повідомлення  StackOverflow.com з’являється в фіді як тег entry, що містить декілька вкладених тегів:

Створення екземпляру Parser

Наступний крок є створити екземпляр парсеру та запустити процесс синтксичного аналізу. В цьому фрагменті парсер ініціалізовано не обробляти простори імен, та використовувати наданий  InputStream як джерело. Він починає поцесс розбору з виклику  nextTag() та викликає метод readFeed(), що виділяє та обробляє дані, в яких зацікавлене застосування:

public class StackOverflowXmlParser {
    // Ми не використовуємо простори імен
    private static final String ns = null;
   
    public List parse(InputStream in) throws XmlPullParserException, IOException {
        try {
            XmlPullParser parser = Xml.newPullParser();
            parser.setFeature(XmlPullParser.FEATURE_PROCESS_NAMESPACES, false);
            parser.setInput(in, null);
            parser.nextTag();
            return readFeed(parser);
        } finally {
            in.close();
        }
    }

Читання фіду

Метод readFeed() робить справжню роботу по обробці фіду. Він шукає елементи, що помічені як "entry" в якості відправної точки для рекурсивної обробки фіду. Якщо тег не entry, він пропускає його. Коли цілий фід був рекурсивно оброблений, readFeed() повертає List, що містить віходження (включаючи вкладені члени даних),  що він виділив з фіду. Цей List потім повертається парсером.

private List readFeed(XmlPullParser parser) throws XmlPullParserException, IOException {
    List entries = new ArrayList();

    parser.require(XmlPullParser.START_TAG, ns, "feed");
    while (parser.next() != XmlPullParser.END_TAG) {
        if (parser.getEventType() != XmlPullParser.START_TAG) {
            continue;
        }
        String name = parser.getName();
        // Починає з пошуку тега tag
        if (name.equals("entry")) {
            entries.add(readEntry(parser));
        } else {
            skip(parser);
        }
    }  
    return entries;
}

Розбір XML

Кроки для розбору XML фіду наступні:

  1. Як описано в Аналіз фіду, визначтесь з тегами, що ви бажаєте включити в ваше застосування. Цей приклад виділяє дані для тегу entry та вкладених тегів title, link та summary.
  2. Створіть наступні методи:

    • Метод "read" для кожного тегу, в якому ви зацікавлені. Наприклад, readEntry(), readTitle(), і таке інше. Парсер читає тегі з вхідного потоку. Коли він зустрічає тег, що називається  entry, title, link або summary, він викликає відповідний метод для цього тегу. Інакше він пропускає тег.
    • Методи для виділення даних для кожного різного типу тегів та для просування парсера до наступного тега. Наприклад:
      • Для тегів title та summary, парсер викликає readText(). Цей метод виділяє дані для ціх тегів викликаючи  parser.getText().
      • Для тегу link, парсер виділяє дані для посилань, спершу визначив, чи є посилання такого типу, що його цікавить. Потім він використовує  parser.getAttributeValue() щоб отримати значення цього посилання.
      • Для тегу entry парсер викликає readEntry(). Цей метод розбирає вкладені теги та повертає об’єкт Entry з членами даних title, link та summary.
    • Допоміжний метод  skip()  є рекурсивним. Для більшої дискусії по цій темі дивіться  Пропущення тегів, які вас не цікавлять.

Цей фрагмент показує, як парсер розбирає entries, titles, links та  summaries.

public static class Entry {
    public final String title;
    public final String link;
    public final String summary;

    private Entry(String title, String summary, String link) {
        this.title = title;
        this.summary = summary;
        this.link = link;
    }
}
  
// Розбирає вміст entry. Якщо зустрічаються теги title, summary або link, віддає 
// обробку до їх відповідних методів "read" для обробки. Інакше пропускаємо тег.
private Entry readEntry(XmlPullParser parser) throws XmlPullParserException, IOException {
    parser.require(XmlPullParser.START_TAG, ns, "entry");
    String title = null;
    String summary = null;
    String link = null;
    while (parser.next() != XmlPullParser.END_TAG) {
        if (parser.getEventType() != XmlPullParser.START_TAG) {
            continue;
        }
        String name = parser.getName();
        if (name.equals("title")) {
            title = readTitle(parser);
        } else if (name.equals("summary")) {
            summary = readSummary(parser);
        } else if (name.equals("link")) {
            link = readLink(parser);
        } else {
            skip(parser);
        }
    }
    return new Entry(title, summary, link);
}

// Обробляє теги title.
private String readTitle(XmlPullParser parser) throws IOException, XmlPullParserException {
    parser.require(XmlPullParser.START_TAG, ns, "title");
    String title = readText(parser);
    parser.require(XmlPullParser.END_TAG, ns, "title");
    return title;
}
  
// Обробляє теги link.
private String readLink(XmlPullParser parser) throws IOException, XmlPullParserException {
    String link = "";
    parser.require(XmlPullParser.START_TAG, ns, "link");
    String tag = parser.getName();
    String relType = parser.getAttributeValue(null, "rel");  
    if (tag.equals("link")) {
        if (relType.equals("alternate")){
            link = parser.getAttributeValue(null, "href");
            parser.nextTag();
        } 
    }
    parser.require(XmlPullParser.END_TAG, ns, "link");
    return link;
}

// Обробляє теги summary.
private String readSummary(XmlPullParser parser) throws IOException, XmlPullParserException {
    parser.require(XmlPullParser.START_TAG, ns, "summary");
    String summary = readText(parser);
    parser.require(XmlPullParser.END_TAG, ns, "summary");
    return summary;
}

// Для тегів title та summary, виділяє їхні текстові значення.
private String readText(XmlPullParser parser) throws IOException, XmlPullParserException {
    String result = "";
    if (parser.next() == XmlPullParser.TEXT) {
        result = parser.getText();
        parser.nextTag();
    }
    return result;
}

}

Пропущення тегів, які вас не цікавлять

Один з кроків в XML розборі вище - це коли парсер пропускає теги, в яких він не зацікавлений. Ось метод  skip()парсера :

private void skip(XmlPullParser parser) throws XmlPullParserException, IOException {
    if (parser.getEventType() != XmlPullParser.START_TAG) {
        throw new IllegalStateException();
    }
    int depth = 1;
    while (depth != 0) {
        switch (parser.next()) {
        case XmlPullParser.END_TAG:
            depth--;
            break;
        case XmlPullParser.START_TAG:
            depth++;
            break;
        }
    }
 }

Ось як він робить:

  • Він викликає виключення, якщо поточна подія не START_TAG.
  • Він споживає START_TAG, та всі події, до та включаючи співпадаючий  END_TAG.
  • Щоб впевнитись, що він зупинився на коректному END_TAG та не на першому тазі, що він зустрів після оригінального START_TAG, він відсліджує глибину вкладення.

Таким чином, якщо поточний елемент має вкладені елементи, значення глибини  depth не буде 0, доки парсер не спожиє всі елементи між оригінальним START_TAG та до його відповідного  END_TAG. Наприклад, уявіть, як парсер попускає елемент <author>, що має два вкладених елементи,  <name> та <uri>:

  • Перший прохід циклу while, наступним тегом, на який натрапить парсер після <author>, буде START_TAG для  <name>. Значення depth збільшиться до  2.
  • Другий прохід while, наступним тегом буде END_TAG </name>. Значення depth зменшится до 1.
  • Третій прохід while, наступний тег  START_TAG <uri>. Значення depth знову 2.
  • Четвертий прохід while, наступний тег END_TAG </uri>. Значення depth зменшено до 1.
  • П’ятий прохід циклу  while , нарешті парсер натрапляє на END_TAG </author>. Значення  depth ззменшується до 0, вказуючи, що елемент  <author> був успішно пропущений.

Споживання даних XML

Застосування-приклад отримує та розбирає XML фід як AsyncTask. Це прибирає обробку з головного потоку UI. Коли обробка завершена, застосування оновлює UI в головній активності (NetworkActivity).

В витягу, показаному нижче, метод loadPage() робить наступне:

  • Ініціалізує рядок змінну в URL для XML фіду.
  • Якщо налаштування користувача та мережеве з’єднання дозволяють це, він викликає  new DownloadXmlTask().execute(url). Це створює екземпляр нового об’єкта DownloadXmlTask (підклас AsyncTask) та виконує його метод  execute(), що завантажує та розбирає фід та повертає рядок результат, що буде зображено в UI.
public class NetworkActivity extends Activity {
    public static final String WIFI = "Wi-Fi";
    public static final String ANY = "Any";
    private static final String URL = "http://stackoverflow.com/feeds/tag?tagnames=android&sort=newest";
   
    // Коли є Wi-Fi з’єднання.
    private static boolean wifiConnected = false; 
    // Коли є мобільне з’єднання.
    private static boolean mobileConnected = false;
    // Коли екран має бути оновлений.
    public static boolean refreshDisplay = true; 
    public static String sPref = null;

    ...
      
    // Використовує AsyncTask для завантаження XML фіду з stackoverflow.com.
    public void loadPage() {  
      
        if((sPref.equals(ANY)) && (wifiConnected || mobileConnected)) {
            new DownloadXmlTask().execute(URL);
        }
        else if ((sPref.equals(WIFI)) && (wifiConnected)) {
            new DownloadXmlTask().execute(URL);
        } else {
            // відобразити помилку
        }  
    }

Підклас AsyncTask відображений нижче, DownloadXmlTask, реалізує наступні методи AsyncTask:

  • doInBackground() виконує метод  loadXmlFromNetwork(). Він отримує URL фіду як параметр. Метод loadXmlFromNetwork() отримує та обробляє фід. Коли він завершується, він повертає результат в вигляді рядка.
  • onPostExecute() приймає повернутий рядок та відображує його в UI.
// Реалізація AsyncTask, що використовується для завантаження XML з stackoverflow.com.
private class DownloadXmlTask extends AsyncTask<String, Void, String> {
    @Override
    protected String doInBackground(String... urls) {
        try {
            return loadXmlFromNetwork(urls[0]);
        } catch (IOException e) {
            return getResources().getString(R.string.connection_error);
        } catch (XmlPullParserException e) {
            return getResources().getString(R.string.xml_error);
        }
    }

    @Override
    protected void onPostExecute(String result) {  
        setContentView(R.layout.main);
        // Displays the HTML string in the UI via a WebView
        WebView myWebView = (WebView) findViewById(R.id.webview);
        myWebView.loadData(result, "text/html", null);
    }
}

Нижче є метод loadXmlFromNetwork(), що викликається з DownloadXmlTask. Він робить наступне:

  1. Створює екземпляр StackOverflowXmlParser. Він створює змінні для List, що зберігає об’єкти Entry  (входження), та title, url, та summary, щоб зберігати значення, що виділені з XML фіду для ціх полей.
  2. Виклики downloadUrl(), що підтягують фід та повертає його як InputStream.
  3. Використовує StackOverflowXmlParser, щоб розібрати InputStream. StackOverflowXmlParser заповнює  List з entries даними з фіду.
  4. Обробляє entries з List, та комбінує дані фіду з розміткою HTML.
  5. Повертає рядок HTML, що відображується в головній активності UI за допомогою AsyncTask через onPostExecute().
// Завантажує XML з stackoverflow.com, розбирає його, та комбінує з розміткою 
// HTML. Повертає рядок HTML.
private String loadXmlFromNetwork(String urlString) throws XmlPullParserException, IOException {
    InputStream stream = null;
    // Створюємо екземпляр парсера
    StackOverflowXmlParser stackOverflowXmlParser = new StackOverflowXmlParser();
    List<Entry> entries = null;
    String title = null;
    String url = null;
    String summary = null;
    Calendar rightNow = Calendar.getInstance(); 
    DateFormat formatter = new SimpleDateFormat("MMM dd h:mmaa");
        
    // Перевіряє, чи користувач встановив бажання бачити підсумковий текст
    SharedPreferences sharedPrefs = PreferenceManager.getDefaultSharedPreferences(this);
    boolean pref = sharedPrefs.getBoolean("summaryPref", false);
        
    StringBuilder htmlString = new StringBuilder();
    htmlString.append("<h3>" + getResources().getString(R.string.page_title) + "</h3>");
    htmlString.append("<em>" + getResources().getString(R.string.updated) + " " + 
            formatter.format(rightNow.getTime()) + "</em>");
        
    try {
        stream = downloadUrl(urlString);        
        entries = stackOverflowXmlParser.parse(stream);
    // Makes sure that the InputStream is closed after the app is
    // finished using it.
    } finally {
        if (stream != null) {
            stream.close();
        } 
     }
    
    // StackOverflowXmlParser повертає List (що називається "entries") об’єктів Entry.
    // Кожний об’єкт представляє одне повідомлення в XML фіді.
    // Цей розділ обробляє список входжень, щоб скомбінувати їх з розміткою HTML.
    // Кожне входження відображується в UI як посилання, що опціонально включає
    // текст підсумку.
    for (Entry entry : entries) {       
        htmlString.append("<p><a href='");
        htmlString.append(entry.link);
        htmlString.append("'>" + entry.title + "</a></p>");
        // Якщо користувач бажає бачити текст підсумків, додаємо його на екран.
        if (pref) {
            htmlString.append(entry.summary);
        }
    }
    return htmlString.toString();
}

// Приймаючи рядок, що є URL, встановлює з’єднання та отримує вхідний потік.
private InputStream downloadUrl(String urlString) throws IOException {
    URL url = new URL(urlString);
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    conn.setReadTimeout(10000 /* milliseconds */);
    conn.setConnectTimeout(15000 /* milliseconds */);
    conn.setRequestMethod("GET");
    conn.setDoInput(true);
    // Починає запит
    conn.connect();
    return conn.getInputStream();
}
