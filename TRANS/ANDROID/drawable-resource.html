Ресурси Drawable

See also

  1. 2D Graphics

Ресурси drawable є головною концепцією для графіки, що може бути намальована на екрані, та яку ви можете отримати за допомогою API, такого як getDrawable(int) або застосувати до іншого XML ресурсу за допомогою атрибутів, таких як  android:drawable та android:icon. Є декілька різних типів drawables:

Файл бітової карти Bitmap
Графічний файл (.png, .jpg або .gif). Створює BitmapDrawable.
Файл Nine-Patch
Файл PNG, що має регіони, що розтягуються, щоб дозволити зміну розміру зображення для вмісту (.9.png). Створює NinePatchDrawable.
Список шарів
Drawable, що створює масив з інших Drawables. Вони малюються в порядку массиву, так що елемент   з більшим індексом буде малюватися зверху. Створює LayerDrawable.
Список станів
Файл XML, що посилається на декілька графічних об’єктів для різних станів (наприклад, щоб використовувати інше зображення для натиснутої кнопки). Створює StateListDrawable.
Список рівнів
Файл XML, що визначає drawable, що керує декількома альтернативних Drawables, кожне присвоєне максимально числове значення. Створює LevelListDrawable.
Перехідний Drawable
Файл XML, що визначає drawable, що може робити перехід між двома drawable ресурсами.Створює TransitionDrawable.
Вставлений Drawable
Файл XML, що визначає drawable, що вставлене в інше drawable на задану відстань. Це корисно, коли View потребує фонове drawble, що меньше ніж дійсні межі View.
Drawable відсікання
Файл XML, що визначає drawable, що відсікає інше Drawable, базуючись на поточному рівні ціх Drawable. Створює ClipDrawable.
Drawable маштабування
Файл XML, що визначає drawable, що змінює розмір іншого Drawable, базуючись на його поточному значенні рівня.  Створює  ScaleDrawable
Drawable фігури
Файл XML, що визначає геометрічну фігуру, включаючи коліри та градієнти. Створює  ShapeDrawable.

Також вивіться документ Ресурси анімації, щоб довідатись, як створювати AnimationDrawable.

Зауваження: Ресурс кольору також може бути використаний в XML. Наприклад, при створенні список станів drawable, ви можете посилатись на ресурс кольору за допомогою атрибута android:drawable (android:drawable="@color/green").

Бітова карта

Зображення бітова карта. Android підтримує файли бітових карт в трьох форматах: .png (переважно), .jpg (прийнятно), .gif (збентежливо).

Ви можете посилатись на файл бітової карти напряму, з використанням імені файлу як ресурсний ID, або створити альтернативний ресурсний ID в XML.

Зауваження: Файли бітових карт можуть бути автоматично оптимізовані за допомогою компрессії без втрат в інструменті aapt на протязі процесу побудови. Наприклад, полвнокольоровий файл PNG, що не потребує більш ніж 256 кольорів, може бути перетворений в 8-бітовий PNG з кольоровою палітрою. Це може призвести до створення зоображення різноцінної якості, але що потребує менше пам’яті. Так що двійникові образи покладені в таку директорію, що може бути змінена на протізі побудови. Якщо ви плануєте читати зображення як бітовий потік, щоб перетворити його на бітову карту, замість цього покладіть ваше зображення в res/raw/, де воно не буде оптимізовано.

Файл бітовой карти

Файл бітової карти це файл .png, .jpg або .gif. Android створює ресурс Drawable для кожного з ціх файлів, коли ви зберігаєте їх в каталозі res/drawable/.

РОЗМІЩЕННЯ ФАЙЛУ:
res/drawable/filename.png (.png, .jpg, or .gif)
Ім’я файлу, що використовується як ресурсне ID.
компільований тип даних ресурсу:
Ресурсне посилання на BitmapDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
приклад:
З зображенням, збереженим як res/drawable/myimage.png, цей XML розміщення застосовує зображення до View: 
<ImageView
    android:layout_height="wrap_content"
    android:layout_width="wrap_content"
    android:src="@drawable/myimage" />

Наступний фрагмент коду отримує зображення як Drawable:

Resources res = getResources();
Drawable drawable = res.getDrawable(R.drawable.myimage);
також дивіться:
  • 2D Graphics
  • BitmapDrawable

Бітова карта XML

Бітова карта XML є ресурсом, визначеним в XML, що посилається на файл бітової карти. Ефектом є псевдоним для необробленого файлу бітової карти. XML може вказувати додаткові сластивості для бітової карти, таке, як розмиття та замощення.

Зауваження: Ви можете використовувати елемент   <bitmap> як потомка елемент  у <item>. Наприклад, коли створюється список станів або список шарівслоїв, ви можете виключити атрибут  android:drawable з елемент  у <item>, та вкласти <bitmap> в нього, що визначає елемент   drawable.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, що використовується як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на BitmapDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<bitmap
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:src="@[package:]drawable/drawable_resource"
    android:antialias=["true" | "false"]
    android:dither=["true" | "false"]
    android:filter=["true" | "false"]
    android:gravity=["top" | "bottom" | "left" | "right" | "center_vertical" |
                      "fill_vertical" | "center_horizontal" | "fill_horizontal" |
                      "center" | "fill" | "clip_vertical" | "clip_horizontal"]
    android:mipMap=["true" | "false"]
    android:tileMode=["disabled" | "clamp" | "repeat" | "mirror"] />
елементи:
<bitmap>
Визначає джерело бітової мапи та її властивості.

атрибути:

xmlns:android
String. Визначає простір імен XML, що мусить бути "http://schemas.android.com/apk/res/android". Це потрібно, тільки якщо <bitmap> є кореневим елеентом — це не потрібно, коли <bitmap> вкладений в <item>.
android:src
Ресурс Drawable. Обов’язкове. Посилання на ресурс drawable.
android:antialias
Boolean. Включає або відключає зглаження.
android:dither
Boolean. Включає або відключає розмиття бітової карти, якщо бітова карта не має того ж конфігурування пікселів,  що й екран (наприклад, : бітова карта ARGB 8888 з екраном RGB 565).
android:filter
Boolean. Enables or disables bitmap filtering. Filtering is used when the bitmap is shrunk or stretched to smooth its apperance.
android:gravity
Ключове слово. Включає силу тяжіння для бітової карти. Тяжіння вказує, де буде розміщуватись бітова карта в своєму контейнері, якщо бітова карта менше, ніж контейнер.

Може бути одним або більше (розділених '|') з наступних значень-констант:

Значення Опис
top Розмістити об’єкт зверху відносно контейнера, не змінюючи розмір.
bottom Розмістити об’єкт знизу відносно контейнера, не змінюючи розмір.
left Розмістити об’єкт з лівого краю відносно контейнера, не змінюючи розмір.
right Розмістити об’єкт з правого  краю відносно контейнера, не змінюючи розмір.
center_vertical Розмістити об’єкт відцентровано по вертикалі відносно контейнера, не змінюючи розмір.
fill_vertical Збільшити вертикальний розмір об’єкту, якщо порібно, щоб повністю заповнити контейнер.
center_horizontal Розмістити об’єкт відцнтровано горізонтально відносно контейнера, не змінюючи розмір.
fill_horizontal Збільшити горізонтальний розмір об’єкту, якщо потрібно, щоб повністю заповнити контейнер.
center Розмістити об’єкт в центрі відносно контейнера в двох, вертикальному та горизонтальному напрямках, не змінюючи розмір.
fill Збільшити горизонтальний та вертикальний розміри об’єкту, якщо потрібно, щоб повністю заповнити контейнер. Це виконується по замовчанню.
clip_vertical Додаткова опція, що може бути встановлена для того, щоб верх та/або низ дочок відсікався по межах контейнеру. Відсікання базується на вертикальному тяжінні: тяжіння top відсікає нижній край, тяжіння bottom відсікає верхній край, та ніяке відсікає обоє.
clip_horizontal Додаткова опція, що може бути встановлена для того, щоб лівий/або правий край дочок відсікався по межах його контейнера. Відсікання базується на горизонтальному тяжінні: ліве тяжіння відсікає правий край, праве тяжіння відсікає лівий край, та ніяке відсікає обоє.
android:mipMap
Boolean. Включає або виключає міпмеп підказку. дивіться setHasMipMap() для більшої інформації. Значення по замовчанню .  false.
android:tileMode
Ключове слово. Визначає режим замощення плиткою. Коли режим замощення включений, бітова карта буде повторюватись. Тяжіння при цьому ігнорується. 

Повинно бути одним з наступних значень констант:

Значенн Опис
disabled Не замощувати бітовою картою. Це значення по замовчанню.
clamp Замінити колір краю, якщо шейдер малює за оригінальними межами
repeat Повторювати зображення шейдеру горизонтально та вертикально.
mirror Повторювати зображення шейдеру горизонтально та вертикально, відзеркалюючи зображення, так що суміжні зображення завжди зшиваються.
приклад:
<?xml version="1.0" encoding="utf-8"?>
<bitmap xmlns:android="http://schemas.android.com/apk/res/android"
    android:src="@drawable/icon"
    android:tileMode="repeat" />
також дивіться:
  • BitmapDrawable
  • Creating alias resources

Nine-Patch

NinePatch є PNG зображенням, в якому ви можете визначити регіони, що розтягуються, що Android маштабує, коли вміст в View перебільшує нормальні межі зображення. Типово ви надаєте цей тип зображення як фон для View, що має щонайменше один вимір встановлений в  "wrap_content", та коли View зростає щоб відповідати вмісту, зображенняNine-Patch також маштабується щоб дорівнювати розміру View. Приклад використання зображення Nine-Patch є фон, використанний для стандартного віджета Android Button, що має розтягуватись, щоб відповідати тексту (або зображення) всередині кнопки.

Так само, як для нормальної бітової мапи, ви можете посилатись на файл Nine-Patch напряму або з ресурсу, визначеному в XML.

Для полвної дискусії як створити файл Nine-Patch з ротягнутими регіонами, дивіться документ 2D Graphics.

Файл Nine-Patch

розташування файлу:
res/drawable/filename.9.png
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на NinePatchDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
приклад:
Беручи зображення, збережене як t res/drawable/myninepatch.9.png, це розміщення XML застосовує Nine-Patch до View: 
<Button
    android:layout_height="wrap_content"
    android:layout_width="wrap_content"
    android:background="@drawable/myninepatch" />
також дивіться:
  • 2D Graphics
  • NinePatchDrawable

XML Nine-Patch

XML Nine-Patch є ресурсом, визначеним в XML, що вказує на файл Nine-Patch. XML може дадавати розмиття для зображення.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на NinePatchDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<nine-patch
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:src="@[package:]drawable/drawable_resource"
    android:dither=["true" | "false"] />
елементи:
<nine-patch>
Визначає джерело Nine-Patch та його властивості.

атрибути:

xmlns:android
String. Обов’язкове. Визначає простір імен XML, що має бути "http://schemas.android.com/apk/res/android".
android:src
Ресурс Drawable. Обов’язкове. Посилання на Nine-Patch.
android:dither
Boolean. Вмикає або вимикає розмиття бітової карти, якщо біто карта не має тієї ж піксельної конфігурації, що й екран (наприклад: бітова карта ARGB 8888 з екраном RGB 565).
приклад:
<?xml version="1.0" encoding="utf-8"?>
<nine-patch xmlns:android="http://schemas.android.com/apk/res/android"
    android:src="@drawable/myninepatch"
    android:dither="false" />

Список шарів

LayerDrawable є об’єкт drawable, що керує масивом інших drawables. Кожний drawable в списку малюється в порядку списку — останній drawable в списку малюється зверху.

Кожний drawable представлений елемент  ом <item> в одному елемент  і <layer-list>.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на LayerDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<layer-list
    xmlns:android="http://schemas.android.com/apk/res/android" >
    <item
        android:drawable="@[package:]drawable/drawable_resource"
        android:id="@[+][package:]id/resource_name"
        android:top="dimension"
        android:right="dimension"
        android:bottom="dimension"
        android:left="dimension" />
</layer-list>
елементи:
<layer-list>
Обов’язкове. Це має бути кореневим елементом. Містить один або більше елемент  ів  <item>.

атрибути:

xmlns:android

Всі елемент  и маштабуються, щоб відповідати розміру зовнішнього View, по замовчанню. Таким чином, ваше зображення в списку шарів в різних позиціях може збільшити розмір Viewта деякі зображення маштабуються, як потрібно. Щоб уникнути маштабування елемент  ів списку, використовуйте елемент   <bitmap> в елемент  і <item> щоб задать drawable, та визначити тяжіння таким чином, що не маштабується, таке як "center". Наприклад, наступний <item> визначає елемент  , що маштабується, щоб розміститися в контейнерному View:

<item android:drawable="@drawable/image" />

Щоб запобігти маштабуванню, наступний приклад використовує елемент   <bitmap> з центрованим тяжінням:

<item>
  <bitmap android:src="@drawable/image"
          android:gravity="center" />
</item>

<?xml version="1.0" encoding="utf-8"?>

    <item>
      <bitmap android:src="@drawable/android_red"
        android:gravity="center" />
    </item>
    <item android:top="10dp" android:left="10dp">
      <bitmap android:src="@drawable/android_green"
        android:gravity="center" />
    </item>
    <item android:top="20dp" android:left="20dp">
      <bitmap android:src="@drawable/android_blue"
        android:gravity="center" />
    </item>
</layer-list>

Зауважте, що цей приклад використовує вкладений елемент   <bitmap>, щоб визначити ресурс Drawable для кожного елеента за тяжінням "center". Це гарантує, що зображення не будуть маштабовані щоб розміщуватись в контейнері, через маштабування, що викликане хсувом зображення.

Це розташування XML застосовує це drawable до View:

    android:layout_height="wrap_content"

    android:src="@drawable/layers" />

Список станів

StateListDrawable є drawable об’єкт, визначений в XML, що використовує декілька різних зображень для представлення тієї ж графіки, в залежності від стану об’єкту. Наприклад, віджет Button може існувати в одному з декількох (нажатий, в фокусі, або ніякий), з використанням drawable списку станів, ви можете провадити різне фонове зображення для кожного стану.

Ви можете описати список станів в XMLфайлі. Кожна графіка, представлена елемент  ом <item> в єдиному елемент  і <selector>. Кожний <item> використовує різні атрибути для опису стану, в якому повинна бути використана графіка для drawable.

Впродовж зміни стану, список станів перебігає зверху донизу, та перший елемент  , що співпадає з поточним станом буде використаний  — вибір не базується на "кращому співпадінні", але просто перший елемент  , що задовільняє мінімальним критеріям стану.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на StateListDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<selector xmlns:android="http://schemas.android.com/apk/res/android"
    android:constantSize=["true" | "false"]
    android:dither=["true" | "false"]
    android:variablePadding=["true" | "false"] >
    <item
        android:drawable="@[package:]drawable/drawable_resource"
        android:state_pressed=["true" | "false"]
        android:state_focused=["true" | "false"]
        android:state_hovered=["true" | "false"]
        android:state_selected=["true" | "false"]
        android:state_checkable=["true" | "false"]
        android:state_checked=["true" | "false"]
        android:state_enabled=["true" | "false"]
        android:state_activated=["true" | "false"]
        android:state_window_focused=["true" | "false"] />
</selector>
елементи:
<selector>
Обов’язкове. Має бути кореневим елементом. Містить один або більше елемент  ів <item> .

атрибути:

xmlns:android
String. Обов’язкове. Визначає простір імен XML, що повинно бути "http://schemas.android.com/apk/res/android".
android:constantSize
Boolean. "true", якщо drawable декларує незмінний внутрішній розмір на протязі зміни стану (розмір є максимальним з усіх станів); "false" якщо розмір варіює на основі поточного стану. По замовчанню false.
android:dither
Boolean. "true", щоб ввімкнути розмиття бітової карти, якщо бітова карта не має ту ж саму піксельну конфігурацію, що і екран (наприклад, бітова карта ARGB 8888, та екран RGB 565); "false" щоб відключити розмиття. По замовчанню true.
android:variablePadding
Boolean. "true", якщо відступи можуть змінюватися, базуючись на поточному стані; "false", якщо відступи повинні оставатись одними й тими ж (базуючись на максимальному відступі всіх станів). Включення цієї можливості потребує щоб ви домовлялись з розміщенням при зміні стану, що часто не підтримується. По замовчанню false.
<item>
Визначає drawable, що слід використовувати впродовж певних станів. Маж бути дитиною елемнету <selector>.

атрибути:

android:drawable
Ресурс Drawable. Обов’язкове. Посилається на ресурс Drawable.
android:state_pressed
Boolean. "true", якщо елемент   повинен використовуватись, коли об’єкт натиснутий (такий, як коли кнопка натиснута/клацнута); "false", якщо елемент має використовуватись по замовчанню, в ненатиснутому стані.
android:state_focused
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт має фокус вводу (як коли користувач обирає ввод тексту); "false" якщо цей елемент має використовуватись по замовчанню, не в фокусі.
android:state_hovered
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт був підсвічений курсором; "false", якщо цей елемент має використовуватись по замовчанню, в непідсвіченому стані. Часто це drawable може бути тим же самим, що використовуєтсья і в стані "focused".

Введено в API рівня 14.

android:state_selected
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт є поточним обраним при навігації за допомогою керівних пристрої’в (як коли відбувається навігація по списку з допомогою клавіш навігації); "false" якщо цей елемент повинен використовуватись, коли об’єкт не є обраний.

обраний стан використовується, коли фокус (android:state_focused) не є достатнім (як коли перегляд списку та елемент   в ньому вибрані за допомогою клавіш навігації).

android:state_checkable
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт можна відмітити; "false" якщо цей елемент повинен використовуватись, коли об’єкт не можна відмітити. (Корисно, тільки якщо об’єкт може перетворюватись з відміченого в той, що не може бути відмічений).
android:state_checked
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт відмічений; "false", ящо він має використовуватись, коли об’єкт не обраний.
android:state_enabled
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт доступний (може сприймати події дотику/); "false", якщо він повинне використовуватись, коли об’єкт відключений.
android:state_activated
Boolean. "true", якщо цей елемент повинен використовуватись, коли об’єкт активований як постійно обраний (як "підсвітка" попередньо обраного елементу списка в перегляді постійної навігації); "false", якщо він має використовуватись, коли цей обє’кт не активований.

Введено в API рівня 11.

android:state_window_focused
Boolean. "true", якщо цей елемент повинен використовуватись, коли вікно застосування  має фокус (застосування на передньому плані), "false", якщо цей елемент повинен використовуватись, коли застосування не має фокусу (наприклад, якщо панель повідомлення спущена вниз, або коли відкритий діалог).

Зауваження: Пам’ятайте, що Android застосовує перший елемент в списку станів, що співпадає з поточним станом об’єкта. Так що, якщо перший елемент в списку не містить жодного з переразованих вище атрибутів, він буде застосовуватись кожного разу, і ось чому ваше значення по замовчанню повинно завжди бути останнім (як демонструє наступний приклад).

приклад:
XML файл, збережений як res/drawable/button.xml: 
<?xml version="1.0" encoding="utf-8"?>
<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:state_pressed="true"
          android:drawable="@drawable/button_pressed" /> <!-- pressed -->
    <item android:state_focused="true"
          android:drawable="@drawable/button_focused" /> <!-- focused -->
    <item android:state_hovered="true"
          android:drawable="@drawable/button_focused" /> <!-- hovered -->
    <item android:drawable="@drawable/button_normal" /> <!-- default -->
</selector>

XML розміщення накладає список станів drawable на Button:

<Button
    android:layout_height="wrap_content"
    android:layout_width="wrap_content"
    android:background="@drawable/button" />

    android:gravity=["top" | "bottom" | "left" | "right" | "center_vertical" |
                     "fill_vertical" | "center_horizontal" | "fill_horizontal" |
                     "center" | "fill" | "clip_vertical" | "clip_horizontal"] />
елементи:
<clip>
Визначає drawable відсічення, має бути кореневим елементом.

атрибути:

xmlns:android
String. Обов’язкове. Визначає простір імен XML, що повинно бути "http://schemas.android.com/apk/res/android".
android:drawable
Ресурс Drawable. Обов’язкове. Посилання на ресурс Drawable, що буде відсічений.
android:clipOrientation
Ключове слово. Орієнтація відсічення.

Повинно бути одним з наступних сталих значень:

Значення Опис
horizontal Відсікати drawable горизонтально.
vertical Відсікати drawable вертикально.
android:gravity
Ключове слово. Вказує, де відсікати в межах drawable.

Повинно бути одним або більше (розділені '|') з наступних сталих значень:

Значення Опис
top Розмістити об’єкт top відносно контейнера, не змінюючи розмір. Коли clipOrientation є "vertical", відсічення проходить по низу drawable.
bottom Розмістити об’єкт знизу відносно контейнера, не змінюючи розмір. Коли clipOrientation є  "vertical", відсічення відбувається по верху drawable.
left Розмістити об’єкт злівого краю відносно контейнера, не змінюючи розмір. Це параметр по замовчанню. Коил clipOrientation є "horizontal", відсічення відбувається по арвому краю drawable. Це значення по замовчанню.
right Розмістити об’єкт зправого краю відносно контейнера, не змінюючи розмір. Коли clipOrientation є "horizontal", відсічення відбувається по левому краю drawable.
center_vertical Розмістити об’єкт відцентровано по вертикалі відносно контейнера, не змінюючи розмір. Відсічення поводиться таким же чином, як коли тяжіння встановлено в "center".
fill_vertical Збільшити вертикальний розмір об’єкта, якщо потрібно, так що від повністю заповнює контейнер. Коли clipOrientation є  "vertical", відсічення не відбувається, оскільки drawable заповнює вертикальний простір (за вінятком коли рівень drawable є 0, і в такому випадку він невидимий).
center_horizontal Розміщує об’єкт відцентровано по горизонталі відносно контейнера, не змінюючи розмір. Відсічення веде себе таким же чином, як коли тяжіння "center".
fill_horizontal Збільшити горизонтальний розмір об’єкту, якщо потрібно, так щоб об’єкт повністю заповнив контейнер. Коли clipOrientation є "horizontal", відсічення не відбувається, оскільки drawable заповнює горизонтальний простір (за винятко коли рівень drawable є 0, і в такому випадку він невидимий).
center Розміщує об’єкт відцентровано відносно контейнера в обох, вертикальному та горизонтальному напрямках, не змінюючи розмір. Коли clipOrientation є  "horizontal"зліва та зправа по горизонталі. Коли clipOrientation є  "vertical", відсічення відбувається зверху та знизу.
fill Збільшити горизонтальний та вертикальний розміри об’єкта, якщо потрібно, так, щоб повністю заповнювати контейнер. Відсічення  не відбувається, оскільки drawable заповнює горизонтальний та вертикальний простір (за винятком, коли рівень drawable є 0, в якому випадку він невидимий).
clip_vertical Додаткова опція, що може бути встановлена,  щоб мати врехній,  та/або нижній край дочірнього елементу відсіченою до границь контейнера. Відсічення базоване на вертикальному тяжінні: верхнє тяжіння відсікає нижній край, нижнє тяжіння відсікає верхній край, та ніякий відсікає обоє.
clip_horizontal Додаткова опція, що може бути встановлена, щоб мати лівий та/або правий край дочірнього елементу відсіченою до границь контейнеру . Відсічення базоване на горизонтальному тяжінні: ліве тяжіння відсікає правий край, лівий відсікає правий край, та ніякий відсікає обоє.

Drawable маштабування

XML, що змінює розмір іншого  drawable, базуючись на поточному рівні.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на ScaleDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<scale
    xmlns:android="http://schemas.android.com/apk/res/android"

                          "fill_vertical" | "center_horizontal" | "fill_horizontal" |

    android:scaleWidth="percentage" />

Drawable фігура

Це базова фігура, визначена в XML.

розташування файлу:
res/drawable/filename.xml
Ім’я файлу, використане як ресурсний ID.
компільований тип даних ресурсу:
Ресурсне посилання на GradientDrawable.
посилання на ресурс:
В Java: R.drawable.filename
В XML: @[package:]drawable/filename
синтаксис:
<?xml version="1.0" encoding="utf-8"?>
<shape
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape=["rectangle" | "oval" | "line" | "ring"] >
    <corners
        android:radius="integer"
        android:topLeftRadius="integer"
        android:topRightRadius="integer"
        android:bottomLeftRadius="integer"
        android:bottomRightRadius="integer" />
    <gradient
        android:angle="integer"
        android:centerX="integer"
        android:centerY="integer"
        android:centerColor="integer"
        android:endColor="color"
        android:gradientRadius="integer"
        android:startColor="color"
        android:type=["linear" | "radial" | "sweep"]
        android:useLevel=["true" | "false"] />
    <padding
        android:left="integer"
        android:top="integer"
        android:right="integer"
        android:bottom="integer" />
    <size
        android:width="integer"
        android:height="integer" />
    <solid
        android:color="color" />
    <stroke
        android:width="integer"
        android:color="color"
        android:dashWidth="integer"
        android:dashGap="integer" />
</shape>
елементи:
<shape>
Це drawable фігури. Це має бути кореневим елементом.

атрибути:

xmlns:android
String. Обов’язкове. Визначає простір імен XML, що повинно бути "http://schemas.android.com/apk/res/android".
android:shape
Ключове слово. Визначає тип фігури. Значення одне з наступних:
Значення Desciption
"rectangle" Прямокутник, що заповнює View. Це фігура по замовчанню.
"oval" Овальна фігура, що заповнює View.
"line" Горізонтальна лінія, що перетинає по ширині View. Ця фігура потребує елемент <stroke>, щоб визначити ширину лінії.
"ring" Фігура коло.

Наступні атрибути використовуються, тільки коли android:shape="ring":

android:innerRadius
Розмірність. Радіус для внутрішнього кола (дірки посередині), як значення розмірності або ресурс розмірності.
android:innerRadiusRatio
Float. Радіус внутрішньої частини, виражене як відноешння до ширини. Наприклад, якщо android:innerRadiusRatio="5", тоді внутрішній радиус дорівнює ширині кола, поділене на 5. Ц езначення перекравається  android:innerRadius. Значення по замовчанню  9.
android:thickness
Розмірність. Товщина кола, як значення розмірності або ресурс розмірності.
android:thicknessRatio
Float. Товищина кола, виражене як співідношення до ширини. Наприклад, якщо android:thicknessRatio="2", тоді товщина дорівнює ширині кола, поділене на  2. Це значення перекривається  android:innerRadius. Значення по замовчанню  3.
android:useLevel
Boolean. "true", якщо використовується з LevelListDrawable. Це повинно нормально бути  "false", або ваше зображення зовсім не з’явиться.
<corners>
Створює скруглені кути фігури. Стосується тільки до прямокутників.

атрибути:

android:radius
Розмірність. Радіус всіх кутів, як значення розмірності або ресурс розмірності. Це перекривається для кожного кута відповідним атрибутом.
android:topLeftRadius
Розмірність. Радіус для верхнього лівого кута, як значення розмірності або ресурс розмірності.
android:topRightRadius
Розмірність. Радіус для верхньго правого кута, як значення розмірності або ресурс розмірності.
android:bottomLeftRadius
Розмірність. Радіус для нижнього лівого кута, як значення розмірності або ресурс розмірності.
android:bottomRightRadius
Розмірність. Радіус для нижнього правого кута, як значення розмірності або ресурс розмірності.

Зауваження: Кожний кут має (з початку) бути впроваджений радіусом, більшим ніж 1, або інакше кути не будуть закруглені. Якщо ви бажаєте, щоб окремий кут не був закруглений, круговим шляхом є використання android:radius, щоб встановити радіус закруглення більший ніж 1, та потім перекрити кожний кут значеннями, що ви дійсно бажаєте, встановлюючи нульове закруглення ("0dp"), якщо ви баєаєте не закруглювати окремий кут.


<gradient>
Задає колір градієнту для фігури.

атрибути:

android:angle
Integer. Кут для градієнту, в градусах. 0 є зліва направо, 90 знизу доверху. Має бути кратним 45. По замовчанню 0.

Float. Відносна позиція X для центру градієнту (0 - 1.0).
android:centerY
Float. Відносна позиція Y для центру градієнту (0 - 1.0).
android:centerColor
Color. Опціональний колір, що іде між початковим та завершальним кольором, як шеснадцятирічне значення або ресурс кольору.
android:endColor
Color. Завершальний колір, як шеснадцятирічне значення або ресурс кольору.
android:gradientRadius
Float. Радіус для градієнту. Застосовується тільки коли android:type="radial".
android:startColor
Color. Початковий колір, як шеснадцятирічне значення або ресурс кольору.
android:type
Ключове слово. Тип градієнтного шаблону, що має застосовуватись. Допустимі значення наступні:
Значення Опис
"linear" Лінійний градієнт. Це значення діє по замовчанню.
"radial" Радіальний градієнт. Початковий колір є центральним кольором.
"sweep" Широкополосний градієнт.
android:useLevel
Boolean. "true", якщо використовується як  LevelListDrawable.
<padding>
Відступ, що застосовується до елементу View (цей відступ є позиціями у вмісті View, не фігури).

атрибути:

android:left
Розмірність. Left відступ, як значення розмірності або ресурс розмірності.
android:top
Розмірність. Top відступ, як значення розмірності або ресурс розмірності.
android:right
Розмірність. Right відступ, як значення розмірності або ресурс розмірності.
android:bottom
Розмірність. Bottom відступ, як значення розмірності або ресурс розмірності.
<size>
Розмір фігури.

атрибути:

android:height
Розмірність. The height of the shape, як значення розмірності або ресурс розмірності.
android:width
Розмірність. The width of the shape, як значення розмірності або ресурс розмірності.

Зауваження: Фыгура маштабуэться до розмыру контейнерного View, проворційно до визначених тут розмірів, по замовчанню. Коли ви використовуєте розмір як ImageView, ви можете обмежити маштабування, встановлюючи  android:scaleType в "center".

<solid>
Суцільний колір, що заповнювати фігуру.

атрибути:

android:color
Color. Колір, що застосовуєтся до фігури, як шеснадцятиричне значення або ресурс кольору.
<stroke>
Рядок контуру для цієї фігури.

атрибути:

android:width
Розмірність. Товщина лінії, як значення розмірності або ресурс розмірності.
android:color
Color. Колір лінії, як шеснадцятиричне значення або ресурс кольору.
android:dashGap
Розмірність. Відстань між штрихами лінії,  як значення розмірності або ресурс розмірності. Працює тільки якщо встановлено android:dashWidth.
android:dashWidth
Розмірність. Розмір кожного штриху лінії, як значення розмірності або ресурс розмірності. Тільки якщо встановлене android:dashGap.
приклад:
XML файл, збережений як res/drawable/gradient_box.xml: 
<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <gradient
        android:startColor="#FFFF0000"
        android:endColor="#80FF00FF"
        android:angle="45"/>
    <padding android:left="7dp"
        android:top="7dp"
        android:right="7dp"
        android:bottom="7dp" />
    <corners android:radius="8dp" />
</shape>

Розміщення XML застосовує drawable фігури до View:

<TextView
    android:background="@drawable/gradient_box"
    android:layout_height="wrap_content"
    android:layout_width="wrap_content" />

Цей код застосування отримує drawable фігури та застосовує його до View:

Resources res = getResources();
Drawable shape = res. getDrawable(R.drawable.gradient_box);

TextView tv = (TextView)findViewByID(R.id.textview);
tv.setBackground(shape);
