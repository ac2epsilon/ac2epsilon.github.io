Швидка підказка: Пишемо ясніший код за допомогою Kotlin SAM перетворень

by Ashraff Hathibelagal7 Aug 2017
Difficulty:BeginnerLength:ShortLanguages:
Android SDKKotlinMobile DevelopmentFunctional ProgrammingJava

Якщо ви є досвідченим розробником застосувань Android, можливо ви використовували балакучий Java 7. Як результат, ви можете винайти стислий синтаксис Kotlin, що орієнтований в напрямку фіункціонального програмування, трохи засмутливим.

Одна з загальних проблем, на яку натрапляють початківці при вивченні Kotlin, є розуміння, як треба робити з Java інтерфейсами, що містять поодинокий метод. Такі інтерфейси всюдисутні в світі Android, та на них часто посилаються як на SAM інтерфейси, де SAM є скороченням до Single Abstract Method.

В цьому короткому туторіалі ми вивчимо все, що вам треба знати, щоб вправно застосовувати Java SAM інтерфейси в коді Kotlin.

1. Що таке SAM перетворення?

Коли ви бажаєте застосувати Java інтерфейс, що містить поодинокий метод в вашому Kotlin коді, ви не маєте вручну створювати клас, що реалізує його. Замість цього ви можете використовувати ліямбда вираз. Дякуючи процесу з назвою SAM конверсія, Kotlin може прозоро конвертувати любий лямбда вираз, чия сигнатура співпадає з такою для єдиного метода інтерфейсу в інтерфейс анонімного класу, що реалізує цей інтерфейс.

Наприклад, розглянемо наступний одно-методовий Java інтерфейс:

1
2
3
public interface Adder {
    public void add(int a, int b);
}

Природний, та Java 7-подібний підхід використання інтерфейсу вище буде включати роботу з виразом object, та буде виглядати приблизно так:

1
2
3
4
5
6
7
// Створення примірника анонімного класу з використанням
// ключового слова object
val adder = object : Adder {
    override fun add(a: Int, b: Int): Int {
        return a + b
    }
}

В значній мірі це непотрібний код, що також одночасно не є дуже читабельним. Змість цього, задіявши можливості Kotlin's SAM конверсії ви можете записати такий еквівалентни код:

1
2
// Створення примірника використовуючи лямбду
val adder = Adder { a, b -> a + b }

Як ви можете бачити, ми замінили анонімний клас коротким лябмда виразом, якому передує ім'я інтерфейсу. Зауважте, що число аргументів лямбда виразу дорівнює числу параметрів в сигнатурі методу інтерфейса.

2. SAM конверсія в викликах фукнцій

Коли робите з Java класами, які мають методи, що сприймають SAM типи в якості аргументів, ви можете далі простити наведений синтаксис. Наприклад, розглянемо наступний Java клас з методом, який очікує об'єкт, що реалізує інтерфейс Adder:

01
02
03
04
05
06
07
08
09
10
11
public class Calculator {
    private Adder adder;
 
    public void setAdder(Adder adder) {
        this.adder = adder;
    }
 
    public void add(int a, int b) {
        Log.d("CALCULATOR", "Sum is " + adder.add(a,b));
    }
}

В вашому Kotlin коді тепер ви можете напряму передати лямбда вираз до метода setAdder(), не ставлячи попереду ім'я інтерфейса Adder.

1
2
val calculator = Calculator()
calculator.setAdder({ a, b -> a+b })

Також, коли ви викликаєте метод, що приймає тип SAM як один аргумент, ви вільні випустити дужки, щоб зробити ваш код навіть ще стислішим.

1
calculator.setAdder { a, b -> a+b }

3. SAM конверсія без лямбда

Якщо ви вважаєте, що лямбда вирази є збентежливими, для вас є гарна новина: SAM конверсії також працюють так само гарно зі звиайними фукнціями.  Наприклад, розглянемо наступну функцію, чия сигнатура співпадає з методом інтерфейсу Adder:

1
2
3
4
5
6
7
fun myCustomAdd(a:Int , b:Int):Int =
    if (a+b < 100)
        -1
    else if (a+b < 200)
        0
    else
        a+b

Kotlin дозволяє вам напряму передавати функцію myCustomAdd() як аргумент до метода setAdder() класу Calculator. Не забувайте посилатись на метод з використанням оператора ::. Ось як це робиться:

1
calculator.setAdder (this::myCustomAdd)

4. Змінна it

В багатьох випадках SAM інтерфейси мають методм з одним параметром. Метод з одним параметром, як слідує з назви, має тільки один параметр у власній сигнатурі. При роботі з такими інтерфейсами Kotlin дозволяє вам оминути параметр в сигнатрі вашої лямбда, та використовувати нефявну змінну на ім'я it в тілі виразу. Щоб прояснити це, розглянемо наступний Java інтерфейс:

1
2
3
public interface Doubler {
    public int doubleIt(int number);
}

При використанні інтерфейсу Doubler в вашому коді Kotlin, ви не маєте явно споминати параметр number в сигнатурі вашого лямбда виразу. Замість цього ви можете посилатись на ньго як на it.

1
2
3
4
5
// Цей лямбда вираз використовує змінну it
val doubler1 = Doubler { 2*it }
 
// це еквівалентно до звичайного лямбда виразу
val doubler2 = Doubler { number -> 2*number }

5. SAM інтерфейси в Kotlin

Як Java розробник, ви можете схилитись до створення SAM інтерфейсів в Kotlin. Однак робити це не є гарною ідеєю. Якщо ви створите SAM інтерфейс в Kotlin, або створите метод Kotlin, що очікує об'єкт, який реалізує SAM інтерфейс, як аргумент, можливість SAM конверсії не буде доступною для вас — SAM конверсія є мехназмом Java-взаємодії, та обмежена тільки Java класами та інтерфейсами.

Оскільки  Kotlin підтримує функції вищих порядків — функції, що можуть приймати інші функції як аргументи — вам ніколи не знадобиться створювати SAM інтерфейси в ньому. Наприклад, якщо клас Calculator буде переписаний в Kotlin, його метод setAdder() може бути записаний так, щоб він напряму приймав функцію як аргумент, замість об'єкта, що реалізує інтерфейс Adder.

1
2
3
4
5
6
7
8
9
class Calculator {
    var adder:(a:Int, b:Int)->Int = {a,b -> 0}
                                    // Реалізація по замовчанню
                                    // Сеттер доступний по замовчанню
 
    fun add(a:Int, b:Int) {
        Log.d("CALCULATOR", "Sum is " + adder(a,b))
    }
}

При використанні наведеного вище класу, ви можете встановити adder в функцію або лямбда вираз з використанням оператора =. Наступний код показує як це робиться:

1
2
3
4
val calculator = Calculator()
calculator.adder = this::myCustomAdd
// АБО
calculator.adder = {a,b -> a+b}

Висновок
