Atom: керівництво з польотів

Глава 1: Розпочнімо

Чому Atom?

Існує багато текстових редакторів. Чому ми маємо витрачати ваш час на навчання щодо використання Atom?

Такі редактори як Sublime та TextMate пропонують зручність, але тільки обмежену розширюваність. З іншого боку спектра  Emacs та Vim пропонують екстреальну розширюваність, але вони не дуже доступні, та можуть бути налаштовані тільки за допомогою скриптових мов спеціального призначення.

Ми вважаємо що можемо зробити краще. Наша ціль є безкомпромісна комбінація зручності та налагодження: редактор, що буде прийнятний для учня початкової школи з першого дня навчання кодуванню, але також інструмент, який вони не переростуть, коли перетворяться на досвідчених хакерів. 

Позаяк ми використовуємо Atom для побудови Atom, те, що почалось як експеримент поступово перетворилось на інструмент, без якого ми не можемо жити. На поверхні Atom є сучасним десктопним текстовим редактором, яу ви і очікували. Але відкрийте капот, та ви знайдете систему, яка благає хакнути себе. 

Ядро Atom

Мережа не без недоліків, але два десятиріччя розробки доробили її в неймовірно підатливу та потужну платформу. Так що коли ми налаштувались зробити текстовий редактор, що ма самі бажали розширювати, веб технології були очевидним вибором. Але спочатку ми мали вивільнити їх від ланцюгів. 

Природний веб

Веб переглядачі чудові для перегляду веб сторінок, вле написання коду є особливою діяльністю, що потребує особливого інструменту. Що більш важливо, переглядач значно обмежує доступ до локальної системи з міркувань безпеки, та для нас текстовий редактор, що не може записувати файли або виконувати локальні підпроцесси був не варіантом. 

З цієї причини ми не будували Atom як традиційне веб застовування. Замість цього Atom є спеціалізованим варіантом Chromium, розробленим бути текстовим редактором, замість бути веб переглядачем. Кожне вікно Atom загалом є локально відтвореною веб сторінкою.

Всі API, доступні для типового застосування Node.js, також доступні для коду, що використовується в JavaScript контексті кожного вікна. Цей гібрид провадить дійсно унікальний досвід розробки з боку клієнта. 

Оскільки все відбуваєтсья локально ви не маєте турбуватись щодо каналів ресурсів, поєднання скриптів та асинхронні визначення модулів. Якщо ви бажаєте завантажити деякий код, просто вкажіть це на початку свого файлу. Система модулів Node спрощує розбиття системи на багато малих, сфокусованих пакунків.

JavaScript, знайомтесь, C++

Взаємодія з природним кодом також справді проста. Наприклад, ми написали обертку навколо рушія регулярних виразів Oniguruma для нашої підтримки граматики TextMate. В переглядачі це б потребувало пригод з NaCl або Esprima. Інтеграція Node робить це простим.

На додаток до Node API ми також надаємо API для природних діалогів, додавання контекстних меню та меню застосування, маніпуляції з розмірами вікна та таке інше. 

Веб технології: гарні новини

Інша гарна річ щодо написання кода для Atom є гарантія, що він виконується нановішій версії Chromium. Це означає, що ми можемо ігнорувати проблеми, такі як сумісність переглядачів та поліфіли. Ми можемо використовувати всі сучасні сяючі можливості веб, вже сьогодні. 

Наприклад, розташування нашого робочого простору та панелей базується на flexbox. Це набираючий силу стандарт, що пройшов через багато змін з часів коли ми почали його використовувати, але нічого з цього не мало значення доки все працювало. 

Маючи цілу індустрію, що просуває вперед веб технологію, ми впевнені, що будуємо Atom на родючому грунті. Природні технології UI приходять та відходять, але веб є стандартом, що стає більш спроможним та  розповсюдженим з кожним минулим роком. Ми захоплені заглиблюватись в його інструменти. 

Текстовий редактор з відкритим кодом

Ми бачимо Atom як чудове доповнення до первинної міссії GitHub побудови кращого програмного забезпечення через спільну роботу. Atom є довгостроковою інвестицією, та GitHub буде продовжувати його розробку за допомоги відданої команди, що просуватиметься вперед. Але ми також знаємо, що самостійно ми не можемо досягти нашого бачення Atom. Як показали Emacs та Vim на протязі останніх трьох десятиріч, якщо ви бажаєте побудувати квітнучу, довготривалу спільноту навколо текстового редактора, він має бути з відкритим кодом. 

Цілий редактор Atom є безкоштовним та з відкритим кодом, та доступний під організацією https://Github.com/atom.

Встановлення Atom на Linux

Щоб встановити Atom на Linux ви можете завантажити Debian package або пакунок RPM або з головного сайта Atom або з сторінки релізу проекта Atom. These packages do not currently have auto-update features, so when you would like to upgrade to a new release of Atom, you will have to repeat this installation process.

Debian та Ubuntu (deb/apt)

Щоб встановити Atom на Debian, Ubuntu, або похідні системи:

# Встановити Atom
sudo dpkg -i atom-amd64.deb

# Встановити залежності Atom, якщо вони відсутні
sudo apt-get -f install
RedHat та CentOS (yum)

Щоб встановити Atom на CentOS, Oracle Linux, RedHat Enterprise Linux, Scientific Linux або пов'язані системи, що використовують менеджер пакунків yum:

sudo yum install -y atom.x86_64.rpm
Fedora (dnf)

Щоб встановити останню версію Atom на Fedora або іншу систему, що використовує менеджер пакунків DNF:

sudo dnf install -y atom.x86_64.rpm
SUSE (zypp)

Щоб встановити останню версію Atom на openSUSE або іншу систему, що використовує менеджер пакунків Zypp:

sudo zypper in -y atom.x86_64.rpm

Портабельний режим

Atom зберігає конфігурацію в директорії .atom, що звичайно розташована в домашній директорії. Однак ви можете виконати Atom в портабельному режимі, коли обоє, застосування та конфігурація зберігаються разом, наприклад, на переносному пристрої зберігання. 

Щоб встановити Atom в портабельному режимі завантажте пакунок zip/tar.gz та розпакуйте його на ваш переносний приклад.

Потім створіть директорію .atom разом в директорії, що містить файли Atom, наприклад:

/media/myusb/atom-1.14/atom
/media/myusb/.atom
Портабельні зауваження
  • Директорія .atom має дозволяти запис
  • Ви можете перемістити існуючу директорію .atom на ваш переносний пристрій
  • Atom може зберігати дані свого користувача Electron в вашій директорії .atom - просто створіть піддиректорію electronUserData в .atom
  • Альтернативно ви можете встановити змінну оточення ATOM_HOME щоб вказати куди побажаєте (ви можете написати скрипт .sh або .cmd для тимчасового встановлення змінної та запуску з цього місця)

 Побудова Atom з першоджерел

Якщо ви тільки бажаєте побудувати Atom з першоджерел, ви також можете зробити це. Репозитарій Atom GitHub має детальні інструкції побудови для Mac, Windows, Linux та FreeBSD.

Налаштування проксі та мережевого екрана

 За мережевим екраном?

Якщо ви за екраном та бачите помилки SSL коли встановлюєте пакунки, ви можете відключити суворий SSL, виконавши таке:

apm config set strict-ssl false
Використовуєте проксі?

Якщо ви використовуєте проксі HTTP(S), ви можете сконфігурувати apm для його використання:

apm config set https-proxy YOUR_PROXY_ADDRESS

Ви можете виконати apm config get https-proxy для перевірки, що все встановлено вірно.

Термінологія

Ви можете знайти всіх різноманітних термінів, що ми використовуємо, через мануал в нашому Глосарії.

Палітра команд

На цьому екрані привітання ми познайомимось з, можливо, найбільш важливою командою в Atom, Палітрою команд (Command Palette). Якщо ви нажмете Ctrl+Shift+P від час відкритої панелі редактора, відкриється палітра команд.

В цій книзі ми будемо використовувати скорочення на кшталт Ctrl+Shift+P для демонстрації виконання команд. Це скорочення по замовчанню для вашої платформи, що була визначена автоматично при завантаженні. 

Якщо ви бажаєте побачити інші платформи, окрім тієї, що була визначена, ви можете обрати інший варіант за допомогою перемикача платформи згори сторінки:

Якщо вибір платформи не присутній, тоді поточна сторінка не має платформ-залежного вмісту. 

Якщо ви перелаштували розкладку Atom, ви можете завжди побачити прив'язки клавіш в Command Palette або на закладинці Keybindings в налаштуваннях Settings View.

Це меню з пошуком може виконати майже довільну з важливіших задач, що доступні в Atom. Замість клацати по всім меню застосування щоб знайти дещо ви можете натиснути Ctrl+Shift+P та знайти команду.

Ви не тільки можете передивитись та швидко знайти серед тисяч можливих команд, але також побачити, чи є асоційована з командою комбінація клавіш. Це дуже добре, бо ви можете знайти шлях до цікавих речей, при цьому вивчаючи послідовності команд щоб зробити це. 

Далі в ций книжці ми спробуємо прояснти як шукати текст в Command Palette на додаток до клавіатурних комбінацій до різних команд.

Налаштування

Atom має декілька налаштувань, які ви можете змінити в перегляді Settings View.

Це включає такі речі, як зміна теми, налаштування переносів, налаштування шрифтів, розмір відступів, швидкість прокрутки та ще багато чого. Також ви можете використовувати цей екран для встановки пових пакунків та тем, що буде розкрите в Пакунках Atom.

Щоб відкрити Settings View ви можете:

  • Використати меню Edit > Preferences
  • Шукайте settings-view:open в Command Palette
  • Використайте комбінацію Ctrl+,
Зміна теми

Перегляд Settings View також дозволяє вам змінити тему Atom. Atom іде з чотирма UI темами, темним та світлим варіантами тем Atom та One, а також іншими вісьма темами. Ви можете змінити тему або встановити нові теми, клацнувши не вкладинці Themes на боковій панелі в Settings View.

Теми UI контролюють стиль UI елементів, такіх, як вкладинки та перегляд дерева, тоді як теми синтаксису контролюють підсвітку синтаксису або тексту, що ви завантажуєте в редактор. Щоб змінити тему синтаксису або UI просто виберіть щось інще в спадаючому списку. 

Також є багато тем на https://atom.io, де ви можете за бажанням обрати щось інше. Ми розкриємо налаштування тем в Зміна тем та створення нової теми в Створення теми.

М'які переноси

Ви можете використати Settings View для вказання налаштувань проміжків та переносів.

Включене "Soft Tabs" буде вставляти проміжки замість справжніх табуляцій, коли ви натискаєте клавішу Tab, та налаштування "Tab Length" вказує, як багато проміжків вставляти, коли ви це робите, або як багато проміжків використовується для продставлення табуляції, якщо "Soft Tabs" відключене.

Опція "Soft Wrap" буде робити перенос рядка, що дуже довгий щоб вмістится в поточному вікні. Якщо м'який перенос відключений рядки будуть просто заходити за край екрана, та вам буде треба прокручувати вікно, щоб побачити залишок вмісту. Якщо "Soft Wrap At Preferred Line Length" перемкнуте, рядки будуть переноситись на 80му символі замість ширини вікна. Ви також можете змінити ширину по замовчанню з 80 символів на цьому ж екрані. 

В Базовому налаштуванні ми побачимо, як встановити різні переваги переносу для різних типів файлів (наприклад, якщо ви бажаєте вімкнути перенос для файлів Markdown, але не для інших ).

Відкриття, зміна та збереження файлів

Тепер, коли ваш редактор виглядає як ви побажаєте, давайте почнемо відкривати та змінювати файли. Це ж текстовий редактор, чи не так?

Відкриття файлів

Є декілька способів для відкриття файлів в Atom. Ви можете зробити це через меню File > Open або натиснувши Ctrl+O, щоб обрати файл зі стандартного диалогу. 

Це корисне для відкриття файлу, що знаходиться не в поточному проекті (більше про це далі), або коли ви бажаєте відкрити нове вікно з якоїсь причини. 

Інший шлях відкрити файл в Atom з командного рядка з використанням команди  atom. Команди atom та apm встановлюються автоматично як частина процессу інсталяції Atom.

Ви можете виконати команду atom з одним або декількома шляхами до файлів, що треба відкрити в Atom.

atom --help
Atom Editor v1.8.0

Usage: atom [options] [path ...]

One or more paths to files or folders may be specified. If there is an
existing Atom window that contains all of the given folders, the paths
will be opened in that window. Otherwise, they will be opened in a new
window.

...

Це чудовий інструмент, якщо ви використовуєте термінал або багато робите в терміналі. Тільки зробіть atom [files] та ви вже готові до редагування.

Редагування та збереження файлів

Редагування файлів є досить прямолінійним. Ви можете клацати та прокручувати за допомогою миші, та набирати для зміни вмісту. Немає особливих режимів редагування або ключових команд. Якщо ви схильні до редакторів з режимами або більш складними ключовими командами, перегляньте перелік пакунків Atom. Є багато пакунків, що емулюють популярні стилі. 

Щоб зберігти файл ви можете обрати File > Save в меню, або комбінацію Ctrl+S. Якщо ви оберете File > Save As або натиснете Ctrl+Shift+S ви можете зберігти поточний вміст під іншим ім'ям. Нарешті, ви можете обрати File > Save All щоб зберігти всі відкриті файли в Atom.

Відкриття директорій

Однак Atom не тільки вміє робити з файлами, напевне ви будете більшість часу робити з проектами з декількох файлів. Щоб відкрити директорію оберіть пункт меню File > Open Folder  та оберіть каталог в диалозі. Ви також можете додати більше однієї директорії до поточного вікна через File > Add Project Folder,  або натискаючи Ctrl+Shift+A.

Ви можете відкрити любу кількість директорій з командного рядка, передаючи їх шляхи до команди atom. Наприклад, ви можете виконати команду atom ./hopes ./dreams, щоб одночасно відкрити директорії hopes та dreams.

Коли ви відкриваєте Atom з однією або декільками директоріями, ви автоматично отримаєте Tree View з боку вашого вікна.

Панель Tree View дозволяє вам переглядати та змінювати структуру файлів та каталогів вашого проекту. Ви можете відкривати, переіменовувати, видаляти та створювати нові файли з цього перегляду.

Ви також можете приховати панель за допомогою Ctrl+\ або команди tree-view:toggle в Command Palette, та Alt+\ перенесе фокус до панелі. Коли панель Tree має фокус ви можете нажати A, M, або Delete, щоб додати, перемістити або видалити файли та теки. Ви також можете клацнути правою кнопкою на файлі або файлі в панелі Tree, щоб побачити декілька різних опцій, включаючи всі перелічені, та на додаток перегляд файлу в природій файловій системі або копіювання шляху в буфер обміну.

Пакунки Atom

Як і більшість частин Atom, панель Tree View не є прямо вбудованою в редактор, але є окремим пакунком, що іде з Atom по замовчанню. Пакунки, що ідуть з Atom, називають Core пакунками. Ті, що не включені в Atom, називаються Community пакунками.

Ви можете знайти першоджерельний код Tree View на GitHub за адресою https://Github.com/atom/tree-view.

Це одна з цікавих властивостей Atom. Багато з його основних можливостей насправді тільки пакунки, реалізовані в той же спосіб, як ви можете реалізувати іншу функціональність. Це означає, що якщо вам, наприклад, не подобається Tree View, ви можете написати вашу власну реалізацію цієї функціональності, та повністю замінити її.

Вікриття файла в проекті

Коли ви відкрили проект в Atom, ви можете просто знайти та відкрити любий файл в цьому проекті. 

Якщо ви натиснете Ctrl+T або Ctrl+P, з'явиться діалог Fuzzy Finder. Це дозволить вам швидко шукати любий файл в вашому проекті, набираючи частини шляху.

Також ви можете шукати тільки наразі вікритих файлів (замість ніж серед всіх файлів в вашому проекті) за допомогою Ctrl+B. Це шукатиме серед ваших "буферів" або відкритих файлів. Ви можете також обмежити цей розпривчастий пошук за допомогою Ctrl+Shift+B, що шукатиме тільки серед файлів, що нові або були модифіковані після вашого останнього Git комміту.

Розпливчастий пошук використовує налаштування конфігурації  core.ignoredNames, fuzzy-finder.ignoredNames та core.excludeVCSIgnoredPaths, щоб фільтрувати файли та теки, що не будуть показані. Якщо ви маєте проект з безлічу файлів, по яких ви не бажаєте робити пошук, ви можете додати шаблони або шляхи до ціх переметрів конфігурації або ваших стандартних файлів .Gitignore. Ми вивчимо більше щодо налаштування конфігурації в Глобальних налаштуваннях конфігурації, але зараз ми можемо просто встановити їх через Settings View під Core Settings.

Обоє, core.ignoredNames та fuzzy-finder.ignoredNames реалізовані як глоб шаблони, як вони реалізовані в модулем minimatch в Node.

Нотація налаштувань комфігурації

Іноді ви побачите посилання на налаштування конфігураціі в повному вигляді, як "Ignored Names in Core Settings". В інших місцях ми використовуємо скорочені імена, як core.ignoredNames. Обоє посилаються на одне і те ж. Скорочення є іменем пакунку, крапка, за яким іде ім'я налаштування в "верблюжому" форматі. 

Якщо ви маєте фразу, та бажаєте перекласти в верблюжий реєстр, слідуйте таким крокам:

  1. Перше слово починаєтсья з малої літери
  2. Всі інші слова починаються з великої літери
  3. Видаліть всі проміжки

Так "Ignored Names" стане "ignoredNames".

Підсумки

Тепер ви маєте базове розуміння того, що таке Atom, та що ви бажаєте з ним робити. Також ви мали встановити його на вашій системі, та бути в змозі використовувати його для найбільш базових операцій редагування тексту.

Тепер ми готові до дослідження цікавих речей.

Використання Atom

Тепер, коли ми розглянули самі основи Atom, ми готові побачити, як насправді почати отримувати більше від його використання. В цій главі ми побачимо, як знайти та встановити нові пакунки щоб додати нову функціональність, а також як встановити нові теми, як робити та маніпулювати текстом більш докладно, як налаштувати редактор за вашими забаганками, як робити з Git для контроля версій і таке інше. 

Наприкінці глави ви повинні мати налаштоване оточення, де вам буде комфортабельно, та ви зможете змінювати, створювати та пересуватись по тексту як майстер.

Пакунки Atom

Ми почнемо з системи пакунків Atom. Як ми вказували раніше, Atom сам по собі є дуже базовим ядром функціональності, що іде з декількома корисними пакунками, як Tree View та Settings View.

Фактично є більше 90 пакунків, що складають всю функціональність, що доступні в Atom по замовчанню. Наприклад, екран Welcome, який ви бачите коли запускаєте Atom, перевірка правопису, теми та Fuzzy Finder всі є пакунками, що обслуговуються окремо, та всі використовують той самий API, до якого ви також маєте доступ, та все це ви побачите в деталях в Хакаючи Atom.

Це означає, що пакунки можуть бути дуже потужними, та можуть змінювати все, від вигляду та відчуття всього інтерфейсу, до базових операцій та навіть базової функціональності. 

Щоб встановити новий пакунок ви можете використовувати вкладинку Install в уже знайомому Settings View. Відкрийте Settings View використовуючи Ctrl+,, клацніть на вкладинці "Install" та наберіть ваш запит пошуку в полі Install Packages.

Пакунки, перелічені тут, були опубліковані на http://atom.io/packages, що є офіціальним реєстром пакунків Atom. Пошук в Settings View буде звертатись до реєстру пакунків Atom та підтягувати все, що задовільняє умовам пошуку. 

Всі пакунки будуть надходити з кнопкою "Install". Клацнувши на ній завантажить та встановить пакунок. Ваш редактор тепер буде мати функціональність, що провадить пакунок.

 Налаштування пакунку

Як тільки ви встановили пакунок в Atom, він буде відображений в Settings View на вкладинці "Packages", разом з всіма попередньо встановленими пакунками, що ішли з Atom. Щоб відфільтрувати список що знайти потрібне ви можете набирати в рядку пошуку прямо під заголовком "Installed Packages".

Клацання на кнопці "Settings" для пакунка перенесе вас до екрана налаштувань для окремого пакунку. Тут ви матиметиме змінити деякі зі змінних по замовчанню для пакунку, передивитись клавіатурні прив'язки, тимчасово відключити пакунок, передивитись першоджерельний код, поцікавитись поточною версією пакунку, сповістити про проблеми та видалити пакунок. 

Якщо виходить нова версія пакунку Atom автоматично визначить це, та ви зможете відновити пакунок або з цього екрана, або з вкладинки "Updates". Це допоможе вам легко підтримувати всі ваші пакунки в актуальному стані. 

 Теми Atom

Ви також можете знайти та встановити нові теми для Atom з Settings View. Це можуть бути або теми UI, або теми синтаксису, та ви можете шукати їх на вкладинці "Install", так само, як ви шукаєте пакунки. Переконайтесь, що ви обрали "Themes" після поля пошуку.

Клацнувши на заголовку теми перенесе вас на сторінку профілю для теми на atom.io, що часто буде мати світлину екрана для теми. Таким чином ви зможете бачити, на що воно схоже, перед тим, як встановити тему.

Клацнувши на "Install" встанвовить тему, та зробить її доступною в списку Theme, як ви бачили в Зміні теми.

 Командний рядок

Пересування в Atom

Хоча пересуватись в Atom досить просто за допомогою миші або клавіш з стрілками, існують деякі комбінації клавіш, що можуть допомогти вам утримувати руки на клавіатурі, та пересуватись трохи швидше.

Atom має підтримку для всіх стандартних пересувань курсора Linux за допомогою комбінацій клавіш. Щоб перейти вверх, вниз, вліво або вправо на один символ, ви можете використовувати стрілки. 

На додаток до пересування на один символ існують декілька парив'язок до клавіш для пересування:

  • Ctrl+Left - Пересунутись до початку слова
  • Ctrl+Right - Пересунутись до кінця слова
  • Home - Пересунутись до першого символа поточного рядка
  • End - Пересунутись до кінця рядка
  • Ctrl+Home - Пересунутись до кінця файлу
  • Ctrl+End - Пересунутись до кінця файлу

Ви також можете перейти напряму до окремого рядка (та стовпчика) за допомогою Ctrl+G. Це перенесе вас до диалогу, де ви можете ввести рядок, до якого треба перейти.  Ви також можете використати синтаксис row:column для стрибка до певного символа в цьому рядку.

 Переход по символам

Також ви можете перестрибнути дещо більш змістовно через Symbols View. Щоб перестрибнути до символу, такому, як визначення методу, натисніть Ctrl+R. Це відкриє перелік всіх символів в поточному файлі, які ви можете фільтрувати, подібно до Ctrl+T. Ви можете також шукати символи у всьому проекті, але це потребує файлу tags.

Ви можете згенерувати файл tags за допомогою утіліти ctags. Коли вона встановлена, ви можете використовувати її для генерації файла tags, виконуючи команду для генерації. Дивіться документацію ctags щодо подробиць.

Коли ви маєте згенерований файл tags, ви можете використовувати його для пошуку символів у всьому вашому проекті за допомогою Ctrl+Shift+R. Це також дозволяє використовувати Alt+Ctrl+Down для переходу, та Alt+Ctrl+Up для повернення від декларації символа під курсором. 

Ви можете налаштувати процес генерації тегів, створивши ваш власний файл .ctags в вашій доомашній директорії, ~/.ctags. Приклад може бути знайдений тут.

Функціональність навігаціі по символах реалізована в пакунку symbols-view.

Закладки

Atom також має чудовий шлях для запам'ятовування окремих рядків в вашому проекті, так до ви можете швидко знайти її. 

Якщо ви натиснете Ctrl+Shift+F2, Atom вімкне "bookmark" на поточному рядку. Ви можете встановити їх для свого проекта, та використовувати для швидкого пошуку та переходу до важливих рядків в проекті. Маленький символ закладки додається до рядка, як на малюнку нижче. 

Якщо ви натиснете F2, Atom буде перестрибувати на наступну закладку в вашому файлі, на якому знаходиться фокус. Якщо ви використаєте Shift+F2 переходи будуть відбуватись в зворотньому напрямку.

Також ви можете передивитись весь список ваших закладок та швидко відфільтрувати їх, або перейти до любої з них, натиснувши Ctrl+F2.

Функціональність закладок реалізована в пакунку bookmarks.


Вибір в Atom

Вибір тексту в Atom підтримує декілька дій, таких, як видалення диапазону, відступ та пошук, та помітка тексту для таких дій, як цитування або огорнення в дужки. 

Виділення копіює багато з команд переміщення. Це насправді ті ж самі комбінації клавіш, що і для переміщення, але з додаванням клавіші Shift.

  • Shift+Up - Вибір нагору
  • Shift+Down - Вибір вниз
  • Shift+Left - Вибрати попередній символ
  • Shift+Right - Вибрати наступний символ
  • Ctrl+Shift+Left - Вибрати до начала слова
  • Ctrl+Shift+Right - Вибрати до кінця слова
  • Shift+End - Вибрати до кінця рядка
  • Shift+Home - Вибрати до першого символа в рядку
  • Ctrl+Shift+Home - Вибрати до початку файла
  • Ctrl+Shift+End - Вибрати до кінця файла

На додаток до команд переміщень курсора також є декілька команд, що дозволяють виділити особливі області вмісту.

  • Ctrl+A - Виділити весь вміст файлу
  • Ctrl+L - Виділити цілй рядок

Редагування та видалення текста

Дотепер ми бачили декілька шляхів для переміщення та вибору регіонів в файлі, так що тепер давайте дійсно змінимо дещо в тексті. Очевидно, ви можете друкувати, щоб вставити символи, але є також декілька шляхів для видалення та муніпуляції текстом, що можуть видатись зручними.  

 Базові маніпуляції

Існує досить багато крутих клавішних комбінацій для базових маніпуляцій текстом, що може стати в нагоді. Їх диапазон від пересування між рядками текста та дублювання рядків, до зміни реєстра.

  • Ctrl+J - Приєднати наступний рядок до кінця поточного рядка
  • Ctrl+Up/Down - Пересуває поточний рядок догори або вниз
  • Ctrl+Shift+D - Дублює поточний рядок
  • Ctrl+K Ctrl+U - Поточне слово підносить в вищий реєстр
  • Ctrl+K Ctrl+L -Поточне слово в нижній реєстр

Atom також має вбудовану функціональність для переформатування параграфа для жорсткого обертання на заданій довжині рядка. Ви можете форматувати поточне виділення, щоб рядки мали не більше 80 (або скільки задане в editor.preferredLineLength) символів, через Alt+Ctrl+Q. Якщо нічого не виділене, буде переформатований поточний параграф.

Видалення та підрізка

Ви також можете видаляти або підрізати текст в вашому буфері за допомогою декількох комбінацій. Будьте безжалісним.

  • Ctrl+Shift+K - Видалити поточний рядок
  • Ctrl+Backspace - Видалити до початку слова
  • Ctrl+Delete - Видалити до кінця слова

Декілька курсорів та виборів

Одна з крутих речей, що може робити Atom прямо з коробки, це підтримка декількох курсорів. Це може бути вкрай корисним для маніпуляцій з довгими списками тексту.

  • Ctrl+Click - Додати новий курсор в місці кліку
  • Alt+Shift+Up/Down - Додати додатковий курсор вище/нижче поточного курсора
  • Ctrl+D - Виділити наступне слово в документі, що те ж саме, що поточне виділене слово
  • Alt+F3 - Виділити всі слова в документі, що такі ж самі, що і наразі виділене слово

Використовуючи ці команди, ви можете розмістити курсори в декількох місцях в вашому документі, та ефективно виконувати ті ж самі команди в декількох місцях одночасно.

Це може бути неймовірно корисним, щоб виконувати багато типів одноманітних завдань, як переіменування змінних, або зміна форматування деякого тексту. Ви можете використовувати це для майже кожного плагіну або команди - наприклад, зміни реєстру та пересування, або дублікації рядків.

Ви можете також використовувати мишу для обрання тексту з нажатою клавішею Ctrl, щоб зробити кілька виборів тексту одночасно.

Проміжки

Atom іде з деякима командами, що дозволяють вам маніпулювати проміжками в вашому документі. Одна дуже корисна пара команд перетворює попередні проміжки в табуляції, та перетворює табуляції в в проміжки. Якщо ви робите з документом, що має змішані проміжки, ці команди чудові для того, щоб впорядкувати файл. Існують прикріплення клавіш для команд проміжків|, так що ви можете пошукати в палітрі команд "Convert Spaces to Tabs" (або навпаки), щоб знайти одну з ціх команд.

Команди проміжків реалізовані в пакункі atom/whitespace. Налаштування для команд проміжків знаходиться на сторінці для налаштування whitespace.

Опція Видаляти заключні проміжки ("Remove Trailing Whitespace") встановлена по замовчанню. Це означає, що кожний раз, коли ви зберігаєте любий файл, що відкритий в Atom, він буде відкидати заключні проміжки в файлі. Якщо ви бажаєте віжключити це, перейдіть до пакунку whitespace в вашій панелі налаштування, та відмініть опцію.

Atom також по замовчанню переконається, що ваш файл має заключний символ нереносу рядка. Ви також можете відключити цю опцію на цьому екрані. 

Дужки

Atom іде з розумною та простою обробкою дужок. 

Вона буде по замовчанню пдісвічувати дужки [], () та  {}, коли ваш курсор знаходиться над ними. Вона також підсвічує спвпадаючі теги XML та HTML.

Atom також буде автоматично доповнювати [], (),  {}, "", '', “”, ‘’, «», ‹›, та зворотні апострофи, коли ви друкуєте відкриваючий символ. Якщо ви маєте вибір, та набираєте один з відкриваючих символів, Atom закриватиме вибір в парні лапки або дужки.

Є декілька інших команд щодо дужок, що ви можете використовувати.

  • Ctrl+M - Переструбнути до дужок, що співпадають з відкриваючими під курсором. Якщо немає відкриваючих дужок, тоді перехід на найближчі закриваючі. 
  • Alt+Ctrl+, - Виділити весь текст в поточних дужках
  • Alt+Ctrl+. - Закрити поточний тег XML/HTML

Функціональність дужок реалізована в пакунку bracket-matcher. Як і всі ці пакунки, щоб змінити замовчання, що мають відношення до обробки дужок, або щоб відключити їх взагалі, ви можете перейти до цього пакунку в Settings.

 Кодування

Atom також іде з деякою підтримкою базового кодування файлів, чи ви бажаєте робити з не-UTF-8 кодованими файлами, або ви бажаєте створити власне кодування.

  • Alt+U - Перемкнути меню для вибору кодування

Якщо ви потягнете диалог кодування, ви можете обрати альтернативне кодування файла, що буде обране, коли ви зберігаєте файл. 

Коли ви відкриваєте файл, Atom спробує автоматично визначити кодування. Якщо Atom не може  визначитись, кодування буде по замовчанню UTF-8, що також є кодуванням по замовчанню для нових файлів.

Якщо ви потягнете меню, та змінете активне кодування на дещо інше, файл буде записаний в цьому кодуванні, коли ви наступний раз будете зберігати його.

Вибір кодування реалізований в пакунку encoding-selector.


Пошук та заміна

Пошук та заміна текста в вашому файлі або проекті швидке та просте в Atom.

  • Ctrl+F - Шукати в буфері
  • Ctrl+Shift+F - Шукати в цілому проекті

Якщо ви запускаєте якусь з ціх команд, ви перейдете в панель Пошук та Заміна (Find and Replace) знизу вашого екрана.

Щоб шукати в поточному файлі, ви можете натиснути Ctrl+F, наберіть рядок полуку, та нажміть Enter (або F3 або кнопку "Find Next") декілька раз, щоб пройти по всім співпадінням в файлі. Панель Find and Replace також містить кнопки для перемикання ігнорування реєстру символів, порівняння регулярних виразів, та поле огляду пошуку для вибраних областей. 

Якщо ви набирете рядок в полі заміщення, ви можете замінити співпвдіння на інший рядок. Наприклад, якщо ви бажаєте замінити кожне співпадіння з рядокм "Scott" на рядок "Dragon", ви маєте ввести ці значення в дів поля ввода, та натиснути кнопку "Replace All", щоб виконати заміщення.

Зауваження: за допомогою пошуку з регулярними виразами, синтаксис посилання на заміни груп пошуку є $1, $2, … $&

Ви також можете знайти та замінити рядки в усьому проекті, якщо ви викличете панель за допомогою Ctrl+Shift+F.

Це гарний спосіб знайти, де в вашому проекті викликається функція, та якір пов'язується до окремого знайденого. Клацніте на співпавшому рядку для переходу на цей рядок в файлі.

Ви можете обмежити пошук до підмножини файлів вашого проекту, через введення глоб шаблону в текстове поле "File/Directory pattern". Наприклад, шаблон  src/*.js обмежить пошук до файлів javascript в каталозі src. Шаблон "глобальних зірочек" (**) може бути використаний для співпадіння з довільною кількістю каталогів. Наприклад, docs/**/*.md буде впівпадати з docs/a/foo.md, docs/a/b/foo.md, та таке інше.

Коли ви маєте декілька відкритих проектів, ця можливість може також бути використана для пошуку тільку в однму з ціх каталогів. Наприклад, якщо ви маєте відкритими теки /path1/folder1 та /path2/folder2, ви можете ввести шаблон, що починається з folder1, щоб шукати тільки в першій теці.

Натисніть Esc, коли вказуєте на панелі Find and Replace, щоб видалити ваш екран від цієї панелі.

Функціональність Find and Replace реалізована в пакунку find-and-replace, та використовує модуль scandal з Node, щоб виконувати справжній пошук.

Клаптики

Клаптики є вкрай потужним шляхом швидко згенерувати загально потрібний код натисканням комбінації клавіш. 

Ідея полягає в тому, що ви друкуєте дещо, як habtm, та потім натискаєте клавішу Tab, та воно розкриваєтсья до has_and_belongs_to_many.

Багато пакунків Core та Community ідуть з пласними специфічними клаптиками. Наприклад, пакунок language-html, що провадить підтримку підсвічування синтаксису HTML, та граматика іде з десятками клаптиків для створення багатьох з різноманітних HTML тегів, що ви можете використовувати. Якщо ви створите новий файл HTML в Atom, ви можете набрати html, та потім натиснути Tab, і воно розшириться до:

<html>
  <head>
    <title></title>
  </head>
  <body>

  </body>
</html>

Також ваш курсор переміститься всередину тега title, так що ви можете безпосередньо почати заповнювати тег. Батато клаптиків мають багато точок фокусу, так що ви можете переміщуватись по ним за допомогою клавішиTab - наприклад, в цьому випадку клаптика HTML, коли ви заповните тег title, ви можете натиснути Tab, та курсор переміститься до середини тегу body.

Щоб побачити всі доступні клаптики для типу файлів, який наразі відкритий, оберіть "Snippets: Available" в Command Palette.

Ви можете також використати fuzzy search для фільтрування цього списку, набиранням тексту в полі пошуку. Обираючи один з них призведе до виконання клаптика там, де знаходиться курсор (або декілька курсорів).

Створення ваших власних клаптиків

Все це досить мило, але що, коли є дещо, що відсутнє в мовному пакунку, або дещо, що треба налаштувати саме для вашого кода? На щастя, додати ваш власний клаптик дуже просто.

В каталозі ~/.atom знаходиться текстовий файл на ім'я snippets.cson, що містить всі ваші власні клаптики, що завантажуються, коли ви завантажуєте Atom. Ви можете просто відкрити цей файл, обравши в меню Edit > Snippets.

Формат клаптика

Так що давайте подивимось на клаптик. Базовий формат виглядає таким чином:

'.source.js':
  'console.log':
    'prefix': 'log'
    'body': 'console.log(${1:"crash"});$2'

Ключі зліва є селекторами, коли ці клаптики мають бути виконані. Найпростіший спосіб щоб визначити, що це може бути, це перейти в мовний пакунок, до якого ви бажаєте додати клаптик, та пошукайте рядок "Scope".

Наприклад, якщо ви бажаєте додати клаптик, що буде робити з файлами Java, ми повинні шукати пакунок language-java в вашому перегляді Settings, та ми побачимо, що Scope є source.java. Потім перед ключем верхнього рівня ставиться крапка (як селектор класа CSS).

Наступний рівень ключів є імена клаптиків. Це використовується для опису клаптика в найбільш читабельний спосіб в меню клаптиків. Ви можете називати клаптики як вам завгодно.

Під кожним іменем клаптика є prefix, що повинен вмикати клаптик, та body, що вставляється, коли клаптик буде вставлений.

Кожний $, за яким іде число, є точка табуляції. Остановки табуляції ідуть по циклу натискань на Tab після вставки клаптика.

Табуляції з однаковим номером створять декілька курсорів. 

Приклад вище додає клаптик log до файлів JavaScript, що буде розширюватись до такого:

console.log("crash");

Рядок "crash" буде зразу виділений, та натискання на табуляцію перемістить курсор після ;

Клавіши клаптика, на відміну від селекторів CSS, можуть бути повторені тульни один раз на одному рівні. Якщо ключі дублюються на деякому рівні, тоді буде прочитаний тільки останній. Дивіться Кофігурація за допомогою CSON щодо додаткової інформації.

Багаторядкове тіло клаптика

Ви також може використовувати багаторядковий синтаксис CoffeeScript з використанням """ для великих шаблонів:

'.source.js':
  'if, else if, else':
    'prefix': 'ieie'
    'body': """
      if (${1:true}) {
        $2
      } else if (${3:false}) {
        $4
      } else {
        $5
      }
    """

Як ви, можливо, очікували, існує клаптик для створення клаптиків. Якщо ви відкриєте файл з клаптиками, наберете snip, та потім Tab, ви отримаєте наступний текст:

'.source.js':
  'Snippet Name':
    'prefix': 'hello'
    'body': 'Hello World!'

просто заповніть поля, та ви отримаєте ваш власний клаптик. Як тільки ви збережете файл, Atom перезавантажить клаптики, та ви безпосередньо будете в змозі використовувати їх.

Декілька клаптиків на одне джерело

Нижче ви можете бачити формат для вставки декількох різних клаптиків в тому ж полі зору в вашому snippets.cson. Просто вставьте ім'я клаптика, префікс, та тіло для додаткових клаптиків в єдиному полі зору. 

'.text.md':
  'Hello World':
    'prefix': 'hewo'
    'body': 'Hello World!'

  'Github Hello':
    'prefix': 'gihe'
    'body': 'Octocat says Hi!'

  'Octocat Image Link':
    'prefix': 'octopic'
    'body': '![GitHub Octocat](https://assets-cdn.Github.com/images/modules/logos_page/Octocat.png)'

Знову, диівіться Конфігураці за допомогою CSON щодо додаткової інформаії по структурі ключів CSON та не-повторюваності.

Додаткова інформація

Функціональність клаптиків реалізована в пакунку snippets.

Для додаткових прикладів дивіться клаптики в пакунках language-html та language-javascript.

Автозавершення

Якщо ви все ще шукаєте, як зберігти декий час при наборі, Atom також іде з  простою функцією автозавершення.

Система автозавершення дозволяє вам переглянути та вставити можливі завершення в редактор, викори стовуючи Tab або Enter.

По замовчанню система буде дивитись в поточному відкритому файлі в пошуках рядка, що ви почали друкувати. 

Якщо ви бажаєте більше опцій, в панелі Settings для пакунка autocomplete-plus ви можете перемикнути налаштування, щоб змусити autocomplete-plus шукати текст в усіх ваших відкритих буферах, скоріше, ніж в поточному файлі. 

Функціональність автозавершення реалізований в пакунку autocomplete-plus.

Згортання

Якщо ви бажаєте побачити огляд структури файла з кодом, з яким ви робите, згортання може бути корисним інструментом. Згортання ховає блоки кода, таких, як функції, або блоки циклів, щоб спростити те, що ви бачите на екрані. 

Ви можете згортати блоки кода, клацаючи на стрілках, що з'являються, коли ви підносите ваш курсор миші над боковиною. Ви також можете згортати та разгортати код з клавіатури за допомогою комбінацій Alt+Ctrl+[ та Alt+Ctrl+].

Щоб згорнути все разом, використовуйте Alt+Ctrl+Shift+[, та розгорнути використовуйте Alt+Ctrl+Shift+]. Ви також можете згорнути до певного рівня відступів за допомогою Ctrl+K Ctrl+0-9, де число є рівнем відступів.

Нарешті, ви можете згорнути довільні розділи вашого кода або текста, зробивши вибір та потім натиснувши Alt+Ctrl+F, або обравши "Fold Selection" в Command Palette.


Панелі

Ви можете розділити панель редактора горизонтально або вертикально, використовуючи Ctrl+K Up/Down/Left/Right, де клавіша напрямку дає напрямок розділення панелі. Коли ви розділили панель, ви можете перемикатись між хми за допомогою Ctrl+K Ctrl+Up/Down/Left/Right, де напрямок є напрямком, куди слід перемістити фокус. 

Кожна панель має власні "панелі", що представлені як закладки. Ви можете переміщувати файли з панелі на панель, через перетягування їх за допомогою миші, та скидання їх на панель, де ви б хотіли бачити свій файл. 

Якщо ви не бажаєте використовувати закладки, ви можете не робити цього. Ви можете відключити пакунок tabs, та кожна панель все ще буде підтримувати декілька примірників панелей. Ви просто не хочете мати закладки, та клацати для перехода між ними. 

Щоб закрити панель, ви можете закрити всі закладки за допомогою Ctrl+W. Ви можете сконфігурувати, чи панелі автоматично закриваються, коли пусті, в налаштуваннях Settings View за допомогою налаштування "Remove Empty Panes" під Core Settings.

Підвислі елементи панелі

"Підвислі елементи панелей" в минулому були відомі як "Preview Tabs"

Коли ви відкриваєте новий файл через клацання в Tree View, буде відкрито нову закладку з заголовком прописом. Це вказує на те, що файл "підвислий". Коли файл підвислий, він буде заміщений наступним підвислим файлом, що буде відкритий. Це дозволяє вам клацати по групі файлів, щоб знайти дещо, не маючи повертатись та закривати їх всі. 

Ви можете підтвердити підвислий файл, зробивши одне з наступного:

  • Двічи клацнути на закладці файла
  • Двічи клацнути на файлі в перегляді дерева каталогів
  • Редагувати вміст файла
  • Зберігти файл

Ви також можете відкрити файл, що вже підтверджений, через подвійне клацання в дереві каталогів, замість одного клацання.

Відключення підвислих елементів панелі

Якщо ви бажаєте не мати файлів, відкритих в підвислому режимі, ви можете відключити цю поведінку, через відключення "Allow Pending Pane Items" в розділі Core Settings на сторінці Settings View. Коли підвислі елементи панелей відключені, одноразове клацання в панелі дерева каталогів будуть виділяти його, але не відкривати. Ви повинні клацнути два раза, щоб відкрити його. 

Граматика

"Граматика" файла є тією мовою, що Atom асоціює з цім файлом. Типи граматики будуть включати "Java" або "GitHub-Flavored Markdown". Ми трохи бачили це, коли створювали деякі клаптики.

Коли ви завантажуєте файл, Atom робить невелику роботу, щоб здогадатись, який це тип файла. Значною мірою це робиться через погляд на розширення файла (.md є файлом Markdown, тощо), хоча іноді він має інспектувати вміст файла, щоб зробити висновок.

Коли ви відкриваєте файл, та Atom не може з'ясувати граматику для цього файла, по замовченню буде самий простий, а саме "Plain Text". Якщо робиться по замовчанню "Plain Text", та це невірна граматика для цього файла, або є якась причина побажати змінити обрану граматику, ви можете витягнути Grammar Selector за допомогою Ctrl+Shift+L.

Коли грамати ка для файла змінена, Atom буде пам'ятати для поточної сессії.

Функціональність Grammar Selector реалізована в пакунку grammar-selector.


Контроль версій в Atom

Контроль версій є важливим аспектом любого проекта, та Atom іде з базовим Git та GitHub.

Щоб використовувати контроль версій в Atom, корень проекта має містити Git репозитарій.

Повернення до HEAD ревізії

Комбінація Alt+Ctrl+Z повертаєтсья до HEAD ревізії файла в редакторі.

Це швидкий спосіб відкинути всі ізбережені та відкладені зміни, що ви зробили, та відновити версію файла до HEAD коміту. В основному це те ж саме, що і виконання Git checkout HEAD -- <path> та Git reset HEAD -- <path> з командного рядка по цьому шляху. 

Ця команда іде через стек відміни, так що ви можете зробити Ctrl+Z після цього, щоб відновити попередній вміст.

Список статуса Git

Atom іде з пакунком fuzzy-finder, що провадить Ctrl+T для швидкого відкриття файлу в проекті, та Ctrl+B для перестрибування до любого відкритого редактора. Пакунок також провадить Ctrl+Shift+B, що відображать список всіх невідсліджуваних та модифікованих файлів в проекті. Це будуть ті самі файли, що ви побачите в командному рядку після запуску Git status.

Зправа кожного фала з'явиться іконка, що покаже, чи файл не відсліджується або змінений.

Writing in Atom

Though it is probably most common to use Atom to write software code, Atom can also be used to write prose quite effectively. Most often this is done in some sort of markup language such as Asciidoc or Markdown (in which this manual is written). Here we'll quickly cover a few of the tools Atom provides for helping you write prose.

In these docs, we'll concentrate on writing in Markdown; however, other prose markup languages like Asciidoc have packages that provide similar functionality.

Spell Checking

If you're working in text (which includes plain text files, GitHub markdown, and Git commit messages by default), Atom will automatically try to check your spelling.

Any misspelled words will be highlighted (by default with a dashed red line beneath the word), and you can pull up a menu of possible corrections by hitting Ctrl+Shift+; (or by choosing "Correct Spelling" from the right-click context menu or from the Command Palette).

To add more types of files to the list of what Atom will try to spell check, go to the Spell Check package settings in your Settings view and add any grammars you want to spell check.

The default grammars to spell check are text.plain, source.gfm, and text.Git-commitbut you can add something like source.asciidoc if you wish to check those types of files too.

The spell checking is implemented in the spell-check package.

Previews

When writing prose in a markup language, it's often very useful to get an idea of what the content will look like when it's rendered. Atom ships with a package for previewing Markdown by default.

  • Ctrl+Shift+M - Will toggle Preview mode for Markdown.

As you edit the text, the preview will also update automatically. This makes it fairly easy to check your syntax as you type.

You can also copy the rendered HTML from the preview pane into your system clipboard. There is no keybinding for it, but you can find it in the Command Palette by searching for "Markdown Preview Copy HTML".

Markdown preview is implemented in the markdown-preview package.

Snippets

There are also a number of great snippets available for writing Markdown quickly.

If you type img and hit tab you get a Markdown-formatted image embed code like ![](). If you type table and hit tab you get a nice example table to fill out.

| Header One     | Header Two     |
| :------------- | :------------- |
| Item One       | Item Two       |

Although there are only a handful of Markdown snippets (b for bold, i for italic, code for a code block, etc), they save you from having to look up the more obscure syntaxes. Again, you can easily see a list of all available snippets for the type of file you're currently in by choosing "Snippets: Available" in the Command Palette.


Basic Customization

Now that we are feeling comfortable with just about everything built into Atom, let's look at how to tweak it. Perhaps there is a keybinding that you use a lot but feels wrong or a color that isn't quite right for you. Atom is amazingly flexible, so let's go over some of the simpler flexes it can do.

Configuring with CSON

All of Atom's config files (with the exception of your style sheet and your Init Script) are written in CSON, short for CoffeeScript Object Notation. Just like its namesake JSON, JavaScript Object Notation, CSON is a text format for storing structured data in the form of simple objects made up of key-value pairs.

key:
  key: value
  key: value
  key: [value, value]

Objects are the backbone of any CSON file, and are delineated by indentation (as in the above example). A key's value can either be a String, a Number, an Object, a Boolean, null, or an Array of any of these data types.

Just like the more common JSON, CSON's keys can only be repeated once per object. If there are duplicate keys, then the last usage of that key overwrites all others, as if they weren't there. The same holds true for Atom's config files.

Don't do this:

# Only the second snippet will be loaded
'.source.js':
  'console.log':
    'prefix': 'log'
    'body': 'console.log(${1:"crash"});$2'
'.source.js':
  'console.error':
    'prefix': 'error'
    'body': 'console.error(${1:"crash"});$2'

Use this instead:

# Both snippets will be loaded
'.source.js':
  'console.log':
    'prefix': 'log'
    'body': 'console.log(${1:"crash"});$2'
  'console.error':
    'prefix': 'error'
    'body': 'console.error(${1:"crash"});$2'

Style Tweaks

If you want to apply quick-and-dirty personal styling changes without creating an entire theme that you intend to publish, you can add styles to the styles.less file in your ~/.atomdirectory. You can open this file in an editor from the Edit > Stylesheet menu.

For example, to change the colors of the Status Bar, you could add the following rule to your styles.less file:

.status-bar {
  color: white;
  background-color: black;
}

The easiest way to see what classes are available to style is to inspect the DOM manually via the Developer Tools. We'll go over the Developer Tools in great detail in the next chapter, but for now let's take a simple look. You can open the Developer Tools by pressing Ctrl+Shift+I, which will bring up the Chromium Developer Tools panel.

With the Developer Tools, you can inspect all the elements in Atom. If you want to update the style of something, you can figure out what classes it has and add a Less rule to your stylesheet to modify it.

If you are unfamiliar with Less, it is a basic CSS preprocessor that makes some things in CSS a bit easier. You can learn more about it at lesscss.org.

If you prefer to use CSS instead, you can do that in the same styles.less file, since CSS is also valid in Less.

Customizing Keybindings

Atom keymaps work similarly to stylesheets. Just as stylesheets use selectors to apply styles to elements, Atom keymaps use selectors to associate key combinations with events in specific contexts. Here's a small example, excerpted from Atom's built-in keymap:

'atom-text-editor':
  'enter': 'editor:newline'

'atom-text-editor[mini] input':
  'enter': 'core:confirm'

This keymap defines the meaning of Enter in two different contexts. In a normal editor, pressing Enter triggers the editor:newline command, which causes the editor to insert a newline. But if the same keystroke occurs inside a select list's mini-editor, it instead triggers the core:confirm command based on the binding in the more-specific selector.

By default, keymap.cson is loaded when Atom is started. It will always be loaded last, giving you the chance to override bindings that are defined by Atom's core keymaps or third-party packages. You can open this file in an editor from the Edit > Keymap menu.

You can see all the keybindings that are currently configured in your installation of Atom in the Keybindings tab in the Settings View.

If you run into problems with keybindings, the Keybinding Resolver is a huge help. It can be opened with the Ctrl+. key combination. It will show you what keys Atom saw you press and what command Atom executed because of that combination.

Глобальні налаштування конфігурації

Atom завантажує конфігурацію з файлуconfig.cson в вашому каталозі ~/.atom.

'*':
  'core':
    'excludeVcsIgnoredPaths': true
  'editor':
    'fontSize': 18

Конфігурація згрупована в глобальні налаштування під ключем *, та мовно-специфічні налаштування під специфічними іменами, як .python.source або .html.text. Під цім ви знайдете налаштування, згуповані по іменем пакунків, або двом базовим просторам імен: core та editor.

You can open this file in an editor from the Edit > Config menu.

Перелік ключів конфігурації
  • core
    • customFileTypes: пов'язує поле зору мови з розширенням (див Налаштування розпізнавання мови)
    • disabledPackages: масив імен пакунків, що відключені
    • excludeVcsIgnoredPaths: не шукати в файлах, відмічених в .Gitignore
    • ignoredNames: імена файлів, що треба ігнорувати в Atom
    • projectHome: директорія, де очікується розташування проектів
    • themes: масив імен тем, в порядку каскадування
  • editor
    • autoIndent: включає або відключає авто-відступи (по замовчанню true)
    • nonWordCharacters: рядок символів не-слів для визначення меж слів
    • fontSize: розмір шрифта редактора
    • fontFamily: сім'я шрифта редактора
    • invisibles: хеш символів, що Atom буде використовувати для малювання проміжків. Ключі є типи символів проміжків, значення є друковані символи (використовуйте значення false для відключення індивідуальних типів проміжків)
      • tab: символи жорсткої табуляції
      • cr: перевод рядка (для закінчень рядка в стилі Microsoft)
      • eol: \n символи
      • space: передуючі та завершаючі проміжки
    • lineHeight: висота рядків редактора, як множник розміру шрифта
    • preferredLineLength: довжина рядка (по замовчанню 80)
    • showInvisibles: чи малювати службові невидимі символи (по замовчанню false)
    • showIndentGuide: показати/приховати індикатори відступів в редакторі
    • showLineNumbers: показати/приховати номери рядків
    • softWrap: вімкнути/вимкнути м'які переноси текста в редакторі
    • softWrapAtPreferredLineLength: вімкнути/вимкнути обертання рядків на  preferredLineLength
    • tabLength: число проміжків в табуляції (по замовчанню 2)
  • fuzzyFinder
    • ignoredNames: файли, що треба ігнорувати тільки в розпливчатому fuzzy-пошуку
  • whitespace
    • ensureSingleTrailingNewline: чи слід скорочувати декілька нових рядків до одного наприкінці файлів
    • removeTrailingWhitespace: вімкнути/вимкнути відкидання проміжків в кінці файлів (по замовченню true)
  • wrap-guide
    • columns: масив хешів pattern та column, що співпадатимуть зі шляхом поточного редактора.

Залежні від мови налаштування файлу конфігурації

Ви також можете редагувати config.cson напряму. Щоб відкрити ваш файл конфігурації через Command Palette, нажміть Ctrl+Shift+P, наберіть open config, та натисніть Enter.

Глобальні налаштування знаходяться в розділі *, та кожна мова може мати свій власний кореневий розділ. Цей розділ є полем огляду мови. Залежні від мови налаштування мають перевагу  над всім в глобальному розділі, але тільки для цієї мови.

'*': # всі мови, окрім перекриттів
  'editor':
    'softWrap': false
    'tabLength': 8

'.source.gfm': # перекриття для markdown
  'editor':
    'softWrap': true

'.source.ruby': # перекриття для ruby
  'editor':
    'tabLength': 2

'.source.python': # перекриття для python
  'editor':
    'tabLength': 4
Пошук імені поля огляду мови

Щоб писати ці перекриття ефективно, вам треба знати ім'я поля огляду для мови. Ми вже робили це для пошуку поля огляду для написання фрагменту в Snippet Format, але ми швидко роскажемо про це знову.

Ім'я поля огляду показане в перегляді налаштування для кожної мови. Клацніть на "Packages" в лівому сайдбарі, найдіть мову, що вас цікавить, виберіть її, та ви повинні побачити ім'я поля огляду під ім'ям мови в заголовку:

Інший шлях знайти поле огляду для специфічної мови є відкрити файл певного типу, та нажати Alt+Ctrl+Shift+P, щоб показати всі поля огляду для поточної позиції під курсором. Поле огляду, що представлене нагорі списку, завжди є мовою для цього типу файла, інші поля огляду специфічні для певної позиції курсора:

These scopes can be especially useful to style the editor, since they can also be used as class names in your stylesheet.

Налаштування розпізнання мови

Якщо ви бажаєте, щоб Atom завжди розпізнавав певні типи файлів в якості певної граматики, вам треба вручну відредагувати ваш файл config.cson. Ви можете відкрити його з використанням команди Open Your Config з командної панелі. Наприклад, якщо ви бажаєте додати розширення foo до мови CoffeeScript, ви можете додати це до вашого файла конфігурації в розділі *.core:

'*':
  core:
    customFileTypes:
      'source.coffee': [
        'foo'
      ]

В прикладі вище source.coffee є ім'я поля огляду (див Пошук імені поля огляду щодо додаткової інформації) та foo є розширенням файлу, що співпадатиме після останньої крапки. Додавання крапки до початку цього поля не буде працювати.

Контроль за тим, де зберігається налаштування Atom

Файли конфігурації CSON для Atom збурігаються на диску вашої машини. Розташування цього місця можна налаштувати. По замовчанню це домашня директорія користувача, що виконує застосунок. Домашня директорія Atom буде, по замовчанню, називатись .atom, та буде розташована в корені домашньої директорії користувча. 

Власне налаштування домівки за допомогою змінної оточення

Змінна оточення може бути використана для переміщення конфігурації Atom в інше розташування. Це можна зробити з декількох міркувань. Одне з них може полягати в тому, що декілька користувацьких рахунків може використовувати одну домівку Atom. Еквівалента змінна, що використовується для задання альтернативного розташування - ATOM_HOME. Якщо ця змінна існує, розташування, вказане в ній, буде використане для завантаження та зберігання налаштувань Atom.

Як взяти ваши налаштування з собою в портабельному режимі

На додаток до використання змінної оточення ATOM_HOME, Atom також може бути встановлений в "портабельному режимі".

Портабельний режим найбільш корисний для того, щоб взяти Atom з собою, з усіма вашими власними налаштуваннями та пакунками, з машини на машину. Це може бути будь-що, від утримання Atom на USB пристрій або на платформу хмарного зберігання, що синхронізується теки з різних машин, як Dropbox. Atom робить в портабельному режимі, коли директорія на ім'я .atom поряд директорії, в якій знаходиться виконуваний файл Atom. Наприклад, інстальована директорія Atom може бути покладена в теку Dropbox, поряд з текою .atom.

При такому налаштуванні Atom буде виконувати ту ж домашню директорію з тими ж налаштуваннями для любої машини, з якою директорія була синхронізована.

Перехід до портабельного режиму

Atom провадить параметр командного рядка для встановлення портабельного режиму (Portable Mode).

atom --portable

Виконання з опцією --portable буде приймати директорію .atom з розташування по замовчанню  (~/.atom) та копіювати відповідний вміст вашої конфігурації в нову директорію розташування портабельного режиму. Це дозволяє просте переміщення з розташування по замовчанню без втрати налаштувань, що ви вже зробили.
