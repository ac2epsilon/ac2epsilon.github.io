Liquid

Безпечна, нагорнута до споживача, мова шаблонів для гнучних веб застосувань.

Що таке Liquid?

Liquid є мовою шаблонів з відкритим кодом, створеню Shopify та написаною на Ruby. Це основа тем Shopify, та використовується для завантаження динамічного вмісту на storefronts.

Liquid був в промисловому режимі з липня 2006, та тепер використовується в багатьох інших коробочних веб застосуваннях.

Хто використовує Liquid?

...та значно більше!


Вступ

Код Liquid може бути категоризований на об'єкти, теги, та фільтри.

Об'єкти

Об'єкти показують, де показувати вміст на сторінці. Імена об'єктів та змінних позначаються подвійними фігурними дужками: {{ та }}.

{{ page.title }}

Introduction

В цьому випадку Liquid відображує вміст об'єкта, названого page.title, та цей об'єкт містить текст Introduction.

Теги

теги створюють лгіку та керують поток для шаблонів. Вони відмічені фігурними дужками зі знаком відсотків: {% and %}.

Маркап, використаний в тегах, не продукує жодного видимого тектсу. Це означає, що ви можете присвоювати змінні та створювати умови та цикли, не відображуючи жодної логіки Liquid.

{% if user %}
  Hello {{ user.name }}!
{% endif %}

Hello Adam!

Теги можуть бути катарегизовані в три типи:

  • Управління потоком
  • Ітерація
  • Присвоєння змінних

Ви можете прочитати більше про кожний тип тегів в відповідних розділах.

Фільтри

Фільтри змінюють вивід об'єкта Liquid. Вони використовуються на виводі, та розділені |.

{{ "/my/fancy/url" | append: ".html" }}

/my/fancy/url.html

Оператори

Liquid включає багато логічних операторів та порівнянь.

Базові оператори

== рівність
!= нерівність
> більше
< меньше
>= більше або рівне
<= меньше або рівне
or логічне АБО
and логічне ТАК

Наприклад:

{% if product.title == "Awesome Shoes" %}
  These shoes are awesome!
{% endif %}

Ви можете використовувати декілька операторів в тезі:

{% if product.type == "Shirt" or product.type == "Shoes" %}
  This is a shirt or a pair of shoes.
{% endif %}

contains

contains перевіряє присутність субрядка в рядку.

{% if product.title contains 'Pack' %}
  This product's title contains the word Pack.
{% endif %}

contains може також перевірити присутність рядка в масиві рядків.

{% if product.tags contains 'Hello' %}
  This product has been tagged with 'Hello'.
{% endif %}

contains може шукати тільки рядки. Ви не можете використати його для перевірки об'єктів в масиві об'єктів.

Правда та неправда

В програмуванні все, що повертає true в умовних виразах називається правдивими. Все що повертає false в умовних виразах називаєтсья неправдивими. Всі типи об'єктів може бути описані як правдіві або неправдиві. 

  • Правдиві
  • Неправдиві
  • Підсумок

Правдиві

Всі значення в Liquid є правдиві, за винятком nil та false.

В прикладі нижче, рядок "Tobi" не є логічною, але вона правдива в контексті порівняння:

{% assign tobi = "Tobi" %}
{% if tobi %}
  This condition will always be true.
{% endif %}

Рядки, навіть пусті, є правдивими. Приклад нижче завершиться пустими HTML тегами, якщо settings.fp_heading пустий:

{% if settings.fp_heading %}
  {{ settings.fp_heading }}
{% endif %}

Неправдиві

Неправдиві значення в Liquid є nil та false.

Типи

Об'єкти Liquid може мати один з шести типів:

  • Рядок
  • Число
  • Логічне
  • Nil
  • Масив

Ви можете ініціалізувати змінні Liquid за допомогою тегів assign або capture.

Рядок

Рядок декларується, оточуючи значення змінної в поодинокі або подвійні лапки:


{% assign my_string = "Hello World!" %}

Число

Числа включають цілі та з плаваючою крапкою:


{% assign my_int = 25 %}
{% assign my_float = 39.756 %}

Логічні

Логічні є або true або false. Лапки не потрібні для визначення логічних значень:


{% assign foo = true %}
{% assign bar = false %}

Nil

Nil є окремим пустим значенням, що повертається, коли код Liquid не має результатів. Це не рядок з символами "nil".

Nil трактується як false в умовах блоківif та інших тегах Liquid, що перевіряє правдивість виразу.

В наступному прикладі, якщо користувач не існує (тобто, user повератє nil), Liquid не буде друкувати привітання:


{% if user %}
  Hello {{ user.name }}!
{% endif %}

Теги або вивід, що повертають nil, не будуть нічого друкувати на сторінці.


The current user is {{ user.name }}


The current user is

Масиси

Масиви утримують списки змінних любого типу.

Присвоєння елементів до масивів

Щоб присвоїти всі елементи до масива, ви можете перебирати їх в циклі в масиві, з використанням тегу ітерації.



{% for user in site.users %}
  {{ user }}
{% endfor %}



Tobi Laura Tetsuro Adam

Доступ до окремих елементів в масиві

Ви можете використовувати нотацію квадратних дужок [ ] для доступу до окремого елемента в масиві. Індекси масивів починаються з нуля. 



{{ site.users[0] }}
{{ site.users[1] }}
{{ site.users[3] }}

Laura
Adam

Ініціалізація масивів

Ви не можете ініціалізувати масиви з використанням тільки Liquid.

Ви можете, однак, використовувати фільтр split для поділу рядка на масив субрядків. 

Керування потоком виконання

Теги керування потоком можуть змінювати інформацію, що показує Liquid, з використанням програмної логіки.

case/when

Створює твердження перемикача, щоб порівняти змінну з різними значеннями. case ініціалізує твердження перемикача, та when порівнює значення.


{% assign handle = 'cake' %}
{% case handle %}
  {% when 'cake' %}
     This is a cake
  {% when 'cookie' %}
     This is a cookie
  {% else %}
     This is not a cake nor a cookie
{% endcase %}


This is a cake

if

Виконує блок кода, тальки якщо певна умова обчислюється до true.


{% if product.title == 'Awesome Shoes' %}
  These shoes are awesome!
{% endif %}


These shoes are awesome!

unless

Протилежність if – виконує блок коду, тільки якщо певна умова не виконується.


{% unless product.title == 'Awesome Shoes' %}
  These shoes are not awesome.
{% endunless %}


These shoes are not awesome.

Це буде еквівалентом зробити наступне:


{% if product.title != 'Awesome Shoes' %}
  These shoes are not awesome.
{% endif %}

elsif / else

Додає більше умов до блокуif або unless.



{% if customer.name == 'kevin' %}
  Hey Kevin!
{% elsif customer.name == 'anonymous' %}
  Hey Anonymous!
{% else %}
  Hi Stranger!
{% endif %}


Hey Anonymous!

Ітерація

Теги Ітерації виконують блок коду декілька разів.

for

Послідовно виконує блок коду. Для повного списку атрибутівциклу for, дивіться forloop (о'бєкт).

Ввод


  {% for product in collection.products %}
    {{ product.title }}
  {% endfor %}

Вивод

hat shirt pants

break

Приводить до переривання циклу коли зустрінеться тегbreak .

Ввод


{% for i in (1..5) %}
  {% if i == 4 %}
    {% break %}
  {% else %}
    {{ i }}
  {% endif %}
{% endfor %}

Вивод

1 2 3

continue

Призводить до того, що цикл пропускає поточну ітерацію, натрапляючи на тегcontinue.

Ввод


{% for i in (1..5) %}
  {% if i == 4 %}
    {% continue %}
  {% else %}
    {{ i }}
  {% endif %}
{% endfor %}

Вивод

1 2 3   5

for (параметри)

limit

Обмежує цикл до вказаного числа ітерацій.

Ввод



{% for item in array limit:2 %}
  {{ item }}
{% endfor %}

Вивод

1 2

offset

Починає цикл з вказаного індексу.

Ввод



{% for item in array offset:2 %}
  {{ item }}
{% endfor %}

Вивод

3 4 5 6

range

Визначає диапазон числе, по яких буде виконуватись цикл. Диапазон може бути визначений обоє, літералами та змінними.

Ввод




{% assign num = 4 %}
{% for i in (1..num) %}
  {{ i }}
{% endfor %}

Вивод

3 4 5
1 2 3 4

reversed

Змінює порядок циклу.

Ввод



{% for item in array reversed %}
  {{ item }}
{% endfor %}

Вивод

6 5 4 3 2 1

cycle

Циклить по групі рядків, та виводить їх в порядку, в якому вони передаються як параметри. Кожного разу, коли викликається cycle, виводиться наступний рядок, переданий як параметр.

cycle має бути використаний в блоці циклу for.

Ввод


{% cycle 'one', 'two', 'three' %}
{% cycle 'one', 'two', 'three' %}
{% cycle 'one', 'two', 'three' %}
{% cycle 'one', 'two', 'three' %}

Вивод

one
two
three
one

Використання для cycle включають:

  • застосування класів парне/непарне до рядків таблиці
  • застосування унікального класа для продукування останньої мініатюри в рядку

cycle (параметри)

cycle сприймає параметер на ім'я cycle group в випадках, коли вам треба декілька блоків cycle в одному шаблоні. Якщо ім'я не надано до групи cycle, тоді вважається, що декілька викликів з тими ж параметрами є одна група.

tablerow

Генерує таблицю HTML. Має бути огорнено в відкриваючий <table> та замикаючий </table> теги HTML.

{% tablerow i in (1..num) %}{{ i }}{% endtablerow %}

Змінні

Тегі змінної створюють нові змінні Liquid.

assign

Створює нову змінну.

Ввод

{% assign my_variable = false %}
{% if my_variable != true %}
  This statement is valid.
{% endif %}

Вивод

  This statement is valid.

Огорніть значення в лапки для утворення рядка.

Ввод

{% assign foo = "bar" %}
{{ foo }}

Вивод

bar

capture

Захоплює рядок в відкриваючому за закриваючому тезі, та присвоює до змінної. Зміні, створені черезе  {% capture %} є рядками.

Ввод

{% capture my_variable %}I am being captured.{% endcapture %}
{{ my_variable }}

Вивод

I am being captured.

increment

Створює ночу числову змінну, та збільшує її на одиницю при кожному виклику. Початкове значення 0.

Ввод

{% increment my_counter %}
{% increment my_counter %}
{% increment my_counter %}

Вивод

0
1
2

Змінні, створені через тег increment, незалежні від змінних, створених через assign або capture.

В прикладі вище зміна на ім'я “var” створюється через assign. Тег increment використовується декілька раз на змінній з тим же ім'ям. Зауважте, що тег increment не впливає на занчення “var”, що було створене через assign.

Ввод

{% assign var = 10 %}
{% increment var %}
{% increment var %}
{% increment var %}
{{ var }}

Вивод

0
1
2
10

decrement

Створює нову змінну, та зменшує її на одиницю при кожному виклику. Початкове значення -1.

Ввод

{% decrement variable %}
{% decrement variable %}
{% decrement variable %}

Вивод

-1
-2
-3

Подібно змінних increment, декларація в decrement незалежна від декларацій через assign або capture.

Фільтри

abs

Повертає абсолютне значення числа.

Ввод

{{ -17 | abs }}

Вивод

17

Ввод

{{ 4 | abs }}

Вивод

4

abs також буде робити з рядками, якщо рядок містить тільки число.

Ввод

{{ "-19.86" | abs }}

Вивод

19.86

append

Конкатенує два рядки та повертає загальне значення.

Ввод

{{ "/my/fancy/url" | append: ".html" }}

Вивод

/my/fancy/url.html

append також може використовуватись зі змінними:

Ввод

{% assign filename = "/index.html" %}
{{ "website.com" | append: filename }}

Вивод

website.com/index.html

capitalize

Вобить перший символ рядка заглавним.

Ввод

{{ "title" | capitalize }}

Вивод

Title

capitalize збільшує тільки перший символ, так що подальші слова не зазнають впливу:

Ввод

{{ "my great title" | capitalize }}

Вивод

My great title

ceil

Округлює ввод нагору до ближцього цілого числа. Liquid намагається конвертувати ввод числа перед застосуванням фільтру.

Ввод

{{ 1.2 | ceil }}

Вивод

2

Ввод

{{ 2.0 | ceil }}

Вивод

2

Ввод

{{ 183.357 | ceil }}

Вивод

184

Ось рядок на вводі:

Ввод

{{ "3.5" | ceil }}

Вивод

4

date

Конвертує часову мітку в інший формат даних. Формат для цього синтаксису такий же, як і в strftime.

Ввод

{{ article.published_at | date: "%a, %b %d, %y" }}

Вивод

Fri, Jul 17, 15

Ввод

{{ article.published_at | date: "%Y" }}

Вивод

2015

date робить на рядках, якщо вони є добре форматованими датами:

Ввод

{{ "March 14, 2016" | date: "%b %d, %y" }}

Вивод

Mar 14, 16

default

Дозволяє вам вказати відкат в випадку, коли значення не існує. default буде показувати своє значення, якщо ліва сторона nil, false, або пуста.

В цьому прикладі product_price не визначений, так що використовується значення по замовчанню.

Ввод

{{ product_price | default: 2.99 }}

Вивод

2.99

В цьому прикладі визначений product_price, так що замовчання не використовується.

Ввод

{% assign product_price = 4.99 %}
{{ product_price | default: 2.99 }}

Вивод

4.99

В уьому прикладі product_price пустий, так що використовується замовчення.

Ввод

{% assign product_price = "" %}
{{ product_price | default: 2.99 }}

Вивод

2.99

divided_by

Ділить число на вказане число.

Результат округлене вниз до найближчого цілого (те саме, що floor).

Ввод

{{ 4 | divided_by: 2 }}

Вивод

2

Ввод

{{ 16 | divided_by: 4 }}

Вивод

4

Ввод

{{ 5 | divided_by: 3 }}

Вивод

1

downcase

Переводить кожний символ до нижнього реєстру.  Для рядків в нижньому реєстрі не має впливу.

Ввод

{{ "Parker Moore" | downcase }}

Вивод

parker moore

Ввод

{{ "apple" | downcase }}

Вивод

apple

escape

Екранує рядок, замінюючи символи екрановою послідовностю (так що цей рядок може бути використаний як URL, наприклад). Він не змінює рядок, що не має щось для екранування.

Ввод

{{ "Have you read 'James & the Giant Peach'?" | escape }}

Вивод

Have you read 'James & the Giant Peach'?

Ввод

{{ "Tetsuro Takara" | escape }}

Вивод

Tetsuro Takara

escape_once

Екранує рядок без зміни ісунючих екрановоних послідовностей. Екрановані рядки не змінюються.

Вхід

{{ "1 < 2 & 3" | escape_once }}

Вихід

1 < 2 & 3

Вхід

{{ "1 < 2 & 3" | escape_once }}

Вихід

1 < 2 & 3

first

Повертає перший елемент списку.

Вхід

{% assign my_array = "apples, oranges, peaches, plums" | split: ", " %}
{{ my_array.first }}

Вихід

apples

Вхід

{% assign my_array = "zebra, octopus, giraffe, tiger" | split: ", " %}
{{ my_array.first }}

Вихід

zebra

floor

Округлює число до низу до найближчого цілого. Liquid намагається перетворити ввод на число при застосуванні цього фільтру.

Вхід

{{ 1.2 | floor }}

Вихід

1

Вхід

{{ 2.0 | floor }}

Вихід

2

Вхід

{{ 183.357 | floor }}

Вихід

183

Ось ввод значення рядка на вході:

Вхід

{{ "3.5" | floor }}

Вихід

3

join

Комбінує елементи масиву в один рядок, використовуючи аргумент як роздільник.

Вхід

{% assign beatles = "John, Paul, George, Ringo" | split: ", " %}

{{ beatles | join: " and " }}

Вихід

John and Paul and George and Ringo

last

Повертає останній елемент масиву.

Вхід

{% assign my_array = "apples, oranges, peaches, plums" | split: ", " %}
{{ my_array.last }}

Вихід

plums

Вхід

truncate

truncate скорочує рядок до числа символів, переданий як параметр. Якщо число вказаних символів менше, ніж довжина рядка, додається (…)  до рядка, та додається до лічильника довжини.
