Ваша перша програма

Традиційно перша програма, що ви напишете на любій програмній мові, називаєтсья “Hello World” – програма, що просто виводить Hello World на ваш термінал. Давайте напишемо її використовуючи Go.

Спершу створіть нову теку, де ви будете зберігати програму. Інсталятор, що ви використовували в главі 1, створив теку в вашій директорії з ім'ям Go. Створіть теку з ім'ям ~/Go/src/golang-book/chapter2. (Де ~ означає вашу домашню теку). В терміналі ви можете зробити це за допомогою наступних команд:

mkdir Go/src/golang-book
mkdir Go/src/golang-book/chapter2

Використовуйте текстовий редактор, та введіть наступне:

package main

import "fmt"

// this is a comment

func main() {
    fmt.Println("Hello World")
}

Переконайтесь, що файл ідентичний до того, що ви бачите тут, та збережіть його як main.go в каталозі, що ви тількі що створили. Відкрийте новий термінал та наберіть наступне:

cd Go/src/golang-book/chapter2
go run main.go

Ви маєте побачити Hello World, що бідображується на вашому терміналі. Команда go run сприймає наступні файли (розділені комами), компілює їх в виконавчий файл в тимчасовому каталозі, та виконує програму. Якщо в и не бачите відображення Hello World, можливо ви допустилися помилки при наборі програми. Компілятор Go надасть вам підказки щодо рядка, де сталася помилка. Як більшість компіляторів, компілятор Go вкрай педантичний та нетерпимий до найменьших помилок. 

Як читати програму

Давайте поглянемо на цю програму більш детально. Програми Go читаються згори до низу, зліва направо (як книжка). Перший рядок каже наступне:

package main

Це відомо як “декларація пакунку”. Кожна програма на Go мусить починатись з декларації пакунка. Пакунки - це шлях Go для організації та повторного викристання коду. Є два типи програм Go: виконавчі та бібліотеки. Виконавчі застосування - це то тип програм, що можуть напряму виконуватись з терміналу (в Windows вони зевершуються на .exe). Бібліотеки - це колекції коду, що ми спакували разом, так що ми можемо використовувати його в інших програмах. Ми дослідиомо це більш детально пізніше, але зараз ми просто переконались, що цей рядок включений в нашу програму.

Наступний рядок пустий. Комп'ютери представляють новий рядок як окремий символ (або декілька символів). Нові рядки, проміжки та табуліція відомі як проміжки (бо ми їх не бачимо). Go здебільшого не турбується про проміжки, ми використовуємо їх щоб спростити читання програм. (Ви можете видалити цей рядок та програма буде поводитись виключно однаково).

Потім ми бачимо це:

import "fmt"

Ключове слово import keyword - це те, як ми вклюаємо код з інших пакунків для використання в нашій програмі. Пакунок fmt (скорочено від format) реалізує форматування для вводу та виводу. З огляду на те, що ми тільки що вивчили щодо пакунків, як ви думаєте, що файли пакунку fmt мають на своєму початку?

Зауважте, що fmt виде закутий в подвійні лапки. Використання подвійних лапок, як тут, відоме як “рядковий літерал”, що є типом “виразу”. В Go рядки представляють послідовність символів (літер, цифр, символів…) визначеної довжини. Рядка описані більш докладно в наступній главі, але зараз важливою річчу буде запам'ятати, що кожний відкритий символ  " мусить десь мати слідуючий інший символ ", та все що між ними є рядком (символ " по собі не є рядком).

Рядок, що починаться з //, відомий як коментар. Коментарі ігноруються компілятором Go, та існують для вашого власного ужитку (або того, хто буде читати первинний код вашої програми). Go підтримує два різні стилі коментарів: коментарі // , в яких весь текст між // та кінцем рядка є частиною коментара, та коментарі /* */ , де все між * є частиною коментара (та може включати декілька рядків).

Після цього подивіться на декларацію функції:

func main() {
    fmt.Println("Hello World")
}

Функції є будівельними блоками програм на Go. Вони мають ввод, вивод, та серію кроків, що називають твердженнями, що виконуються послідовно. Всі функції починаються з ключового слова func , за яким слідує ім'я функції (в данному випадку main), список з нуля або більше “параметрів” в дужках, опціональний тип значення, та “тіло”, що заточене в фігурні дужки. Ця функція не має параметрів, нічого не повертає, та має тільки одне твердження. Ім'я main особливе, бо це функція, що викликається, коли виконується програма.

Фінальний шматок програми наступний:

    fmt.Println("Hello World")

Це твердження складається з трьох компонент. Спершу ми отримуємо доступ до іншої функції з пакунку fmt за назвою Println (це частина fmt.Println, Println означає Print Line). Потім ми створюємо новий рядок, що містить Hello World та викликаємо (або ще кажуть виконуємо) цю функцію з рядком в якості першого та єдиного аргументу. 

На цей момент ми вже побачили багато нової термінології, та ви можете бути трохи спантеличені. Іноді корисно навмисне прочитати вашу прогруму вголос. Одне з прочитань нашої програми, що ми тільки що написали, може бути таким:

Створити нову виконавчу програму, що посилаєтся на бібліотеку fmt та містить одну функцію, що називається main. Ця функція не сприймає аргументи, нічого не повертає, та робить наступне: отримує доступ до фукнції Println що міститься в пакунку fmt , та викликає її з одним вргументом, рядком Hello World.

Функція Println виконує дійсну роботу вашої програми. Ви можете знайти більше про неї, набравши наступне в вашому терміналі:

godoc fmt Println

Проміж іншого ви повинні побачити таке:

Println formats using the default formats for its operands and writes to standard output. Spaces are always added between operands and a newline is appended. It returns the number of bytes written and any write error encountered.

Go є дуже добре документованою мовою програмування, але її документацію може бути складно зрозуміти, якщо ви не дуже знайомі з мовами програмування. Однак команда godoc екстремально корисна, та є добри місцем для початку, якщо в вас виникають питання.

Повертають до функції, що ми розглядаємо, ця документація каже вам, що функція Println буде надсилати що ви їй скажете до стандартного виводу – їм'я для виводу на термінал, в якому ви працюєте. Ця функція саме те, що відображує Hello World.

В наступній главі ви дослідимо, як Go зберігає та представляє речі, такі як Hello World , та вивчимо що таке типи.

Проблеми

  • Що таке проміжки?

  • Що таке коментар? Які два методи існують для запису коментарів?

  • Наша програма починається з package main. З чого починаються файли в пакунку fmt?

  • Ми використали функцію Println, визначену в пакунку fmt. Якщо ми бажаємо функцію Exit з пакунку os, що ми маємо робити?

  • Модифікуйте програму, що ми написали, так, щоб вона замість Hello World друкувала Hello, my name is з вказанням ім'я.
