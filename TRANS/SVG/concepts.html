2.1 Пояснення назви: SVG

SVG означає Масштабована Векторна Графіка (Scalable Vector Graphics), граматика XML для стилізованої графіки, корисна в якості простору імен XML.

Масштабована

Бути масштабованою означає підвищувати або знижувати однорідність. В термінах графіки, масштабована означає не бути обмеженою одним, фіксованим розміром пікселя. В веб, масштабована означає, що певна технологія може зростати до великого числа файлів, великого числа користувачів, широкого різноманіття застосувань. SVG, що є графічна технологія для веб, є масштабованою в обох сенсах цього слова.

SVG графіка масштабована для різних розподільних можливостей екранів, так що, наприклад, друкований вихід використовує повну розподільну здатність друкарського обладнання, та може бути відтворена з тим же розміром на екранах з відмінною розподільчою здатністю. Та ж SVG графіка може бути розміщена на різних розмірах тієї ж веб сторінки, та повторно використана в різних розмірах на різних сторінках. SVG графіка може бути збільшена, щоб роздивитись малі деталі, або щоб допомогти тим, хто погано бачить.

SVG графіка є масштабованою, тому що той же вміст SVG може бути окремою графікою, або може бути посиланням або ж включеним в іншу SVG графіку, таким чином дозволяючи, щоб складні ілюстрації біли побудовані по частинах, можливо різними людьми. Можливості символів, маркерів та шрифтів просувають повторне використання графічних компонент, максимізує переваги HTTP кешування та запобігає потребі в централізованому регістрі схвалених символів.

Вектор

Векторна графіка містить геометричні об'єкти як лінії та криві. Це надає більшої гнучкості, порівняно з тільки растровими форматами (як PNG та JPEG), що мають зберігати інформацію для кожного пікселя зображення. Зазвичай, векторні формати можуть також інтегруватися з растровими зображеннями, та можуть поєднувати їх з векторної інформацією, такою, як шляхи відсічення, для продукування завершеної ілюстрації; SVG не є виключенням в цьому сенсі.

Оскільки всі сучасні екрани є орієнтованими на растр, розбіжність між тільки растровою та векторною графікою полягає тільки в тому, де вони растризовані; це виконує клієнтська частина в випадку векторної графіки, на відміну від вже растризованої на сервері. SVG надає контроль над процесор растризації, наприклад, щоб дозволити згладжування витворів мистецтва, без огидних візуальних перешкод, що типові для низькоякісних векторних реалізацій. SVG також надає растрові ефекти фільтрації з боку клієнта, тому перехід до векторного формату не означає втрату популярних ефектів, таких як м'яко падаючи тіні.

Графіка

Більшість існуючих граматик XML представляють або текстуальну інформацію, або представляють голі дані, такі, як фінансова інформація. Вони типово провадять тільки рудиментарні графічні можливості, часто менше здібні, чим елемент HTML 'img'. SVG заповнює проміжок на ринку, впроваджуючи багате, структуроване визначення для векторної та змішаної, векторно-растрової, графіки; вона може бути використана окремо, або як простір імен XML разом з іншими граматиками.

XML

XML, як для структурованого обміну інформацією, набув шаленої популярності, та є і широко, і надійно реалізованим. Будучи записаним в XML, SVG будується на цих міцних засадах, і отримує багато переваг, таких, як міцна основа для інтернаціоналізації, потужні можливості для структурування, об'єктна модель, і таке інше. Будуючись на існуючих, зрозуміло реалізованих специфікаціях, базовані на XML граматики є відкритими для реалізації без величезних зусиль зворотнього інжинірінгу.

Простір імен

Звичайно, буде корисним мати окремий, тільки для SVG, переглядач. Але SVG також призначено для використання як компонент в багато-просторовому застосуванні XML. Це помножує потужність кожного з використаних просторів імен, та дозволить створювати новий, інноваційний зміст. Наприклад, графіка SVG може бути включена в документ, який використовує тільки орієнтований на текст простір імен XML - включаючи XHTML. Науковий документ, наприклад, може також використовувати MathML для математичних частин документу. Комбінація SVG та SMIL призводить до цікавих, основаних на часі, багатих графічних презентацій.

SVG є гарним компонентом загального призначення для кожної багато-просторової граматики, що потребує графіки.

Стилізація

Переваги листів стилів в термінах контролю за презентацією, гнучкості, швидшого завантаження та покращеного обслуговування на тепер є загально прийнятними, звичайно, при використанні з текстом. SVG поширює цей контроль на царину графіки.

Комбінацію скриптів, DOM та CSS часто позначають терміном як "Динамічний HTML" та широко використовують для анімації, інтерактивності та ефектів презентації. SVG дозволяє такі ж основані на скриптах маніпуляції з деревом документу та листами стилів.

2.2 Важливі концепції SVG

Графічні об'єкти

Як з кожною XML граматикою, були прийняті рішення щодо того, що саме буде моделюватися. Для текстових форматів моделювання зазвичай є на рівні параграфів та фраз, скоріше чим окремих іменників, прислівників або фонем. Подібним чином SVG моделює графіку на рівні графічних об'єктів, скоріше ніж окремих точок.

SVG провадить головний елемент шляху, що може бути використаний для створення безлічі різних графічних об’єктів, а надає загальні базові фігури, таки як прямокутники та еліпси. Вони зручні для ручного кодування там можуть бути застосовані у такій же спосіб, що і більш загальний елемент шляху. SVG надає тонкий контроль над системою координат, в якій визначаються графічні об’єкти, та за перетвореннями, які будуть застосовані протягом малювання.

Символи

Буде можливим визначити деякі стандартні символи, що SVG буде надавати. Але саме які? Завжди будуть додаткові символи для електроніки, картографії, блок-схем, таке інше, що їх будуть потребувати люди, та що не будуть надані до “наступної версії”. SVG дозволяє користувачам створювати, повторно використовувати та поширювати свої власні символи, без потреби в централізованому реєстрі. Спільноти користувачів можуть створювати та покращувати символи, які їм потрібні, без потреби питати дозволу в комітету. Дизайнери можуть точно впевнені в графічному вигляді символів, які вони використовують, та не хвилюватися, що символ може виявитись не підтримуваним.

Символи можуть бути використані з різними розмірами та в різних орієнтаціях, та можуть отримати змінений стиль, щоб підходити до решти графічної композиції.

Растрові ефекти

Багато з існуючих веб графіків використовують операції фільтрації, що можна знайти в графічних пакетах, щоб створити розмиття, тіні, ефекти блискавки, і таке інше. За допомогою растеризації на стороні клієнта, що використовується в векторній графіці, таки ефекти можуть здаватись неможливими. SVG дозволяє декларативне задання фільтрів, або по одному, або в комбінації, що може бути застосовано на стороні клієнта, в помент малювання SVG. Вони задаються таким чином, що графіка постійно зостається масштабованою, та може зображуватись в різних розподільних якостях.

Шрифти

Графічно багатий матеріал часто дуже залежить від окремого використовуваного шрифту та точних проміжків між гліфами. В багатьох випадках дизайнери перетворюють текст на контури для запобігання жодних проблем з підміною шрифту. Це означає, що начальний текст не присутній, а значить страждає можливість пошуку та доступності. У відповідь на запит дизайнерів, SVG включає елементи шрифту, і таким чином зберігаються як текстове, так і графічне представлення.

Анімація

Анімація може бути продукована через скриптові маніпуляції з документом, але скрипти важкі для редагування, та взаємодія між різними інструментами стає ускладнена. Знову, у відповідь на запит від спільноти дизайну, SVG включає елементи декларативної анімації, що були розроблені сумісно SVG та SYMM Working Groups. Це дозволяє анімаційним ефектам, звичайним в існуючій веб графіці, бути вираженими в SVG.

2.3 Можливості використання SVG на веб сторінках

Є різноманітні шляхи, якими вміст SVG може бути включений в веб сторінку. Ось деякі з можливостей:

Окрема веб сторінка SVG
В цьому випадку документ SVG (тобто, веб ресурс, чий MIME тип є "image/svg+xml") завантажується напряму в користувацький агент, такий, як веб переглядач. Документ SVG є веб сторінкою, що презентується користувачеві.
Вбудовування за посиланням
В цьому випадку батьківська веб сторінка посилається на окремо збережений SVG документ, та вказує, що цей SVG документ має бути вбудовано в якості компонента батьківської веб сторінки. Для HTML або XHTML є три можливості:
  • Елемент HTML/XHTML ‘img’ є найбільш загальним методом для використання графіки на HTML сторінках. Для швидшого відображення ширина та висота зображення можуть бути передані як атрибути. Один атрибут, що є обов'язковим, є ‘alt’, що використовується для надання альтернативного текстового рядка для людей, що переглядають сторінки з відключеними зображеннями, або для тих, хто не може бачить зображень. Рядок не може включати жодної розмітки. Атрибут ‘longdesc’ дозволяє вам вказати на більший альтернативний текст — часто в вигляді HTML — що може містити розмітку та багатше форматування.
  • Елемент HTML/XHTML ‘object’ может містити інші елементи, вкладені в нього, на відміну від ‘img’, що є пустим. Це означає, що декілька різних форматів можуть бути запропоновані, використовуючи вкладені елементи ‘object’, з фінальною текстовою альтернативою (включаючи розмітку, посилання, тощо). Буде використано найбільш зовнішній елемент, що може бути відображено.
  • Елемент HTML/XHTML ‘applet’, що може викликати аплет Java для перегляду вмісту SVG на даній веб сторінці. Ці аплети можуть робити багато речей, але їх загальним використанням є відображення зображень, особливо в незвичайних форматах, або коли зробити презентацію під контролем програми з інших міркувань.
Вбудовування одним рядком
В цьому випадку, вміст SVG вбудовується одним рядком безпосередньо напряму в батьківську веб сторінку. Прикладом є XHTML веб сторінка, з фрагментом документу SVG, текстуально включеним в XHTML.
Зовнішнє посилання, з використанням елементу ‘a’ HTML
Це дозволяє бути використаному будь-якому окремому переглядачу SVG, що може (але не має) бути іншою програмою, чим та, що використовується для перегляду HTML. Ця можливість типово використовується для незвичних форматів зображень.
Посилання з властивостей CSS або XSL
Коли користувацький агент підтримує CSS стилізований XML вміст [CSS2] або XSL [XSL], та користувацький агент є Сумісним переглядачем SVG, тоді цей користувацький агент має підтримувати можливість посилання на SVG ресурси з CSS чи XSL властивостей, дозволяючи посилання растрових зображень, включаючи можливість замощування SVG графікою, коли це необхідно, та можливість композиції SVG в фоні, якщо він має прозорі частини. Приклади включають властивості ‘background-image’ та ‘list-style-image’ ([CSS2], розділи 14.2.1 та 12.6.2), що включені в обох, CSS та XSL.
