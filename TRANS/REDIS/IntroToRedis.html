*Вступ до Redis

Redis є покращеним кешем ключ-значення та системою зберігання з відкритим кодом та ліцензією BSD. На нього часто посилаються як на сервер структурованих даних, осклільки він може містити рядки, хеші, списки, множини, сортовані множини, бітові карти та гіпержурнали.

Ви можете виконувати атомарні операції на ціх типах, як додавання до строки; збільшення значення в хеші; затовкування елементу в список; обчисленя перетину множин, об’єднання та різниці; або отримання члену з найвищим рангом в сортованій множині.

Що досягти виняткової продуктивності, Redis працює з наборами даних в пам’яті. В залежності від вашого випадку, ви можете зберігати дані або через дампи даних на диск часом від часу, або додаванням кожної команди до журналу. Зберігання може бути опціонально відключене, якщо все що вам потрібне, це багатий на можливості, мережевий кеш в пам’яті.

Redis також підтримує трівіальну в установці асинхронну реплікацію майстер-підлеглий, з дуже швидкою неблокуючою першою синхронізацією, авто-з’єднанням з частковою синхронізацією після розриву мережі.

Інші можливості включають:

  • Транзакції
  • Підписки Pub/Sub
  • Скрипти Lua
  • Ключі з лімітованим часом життя
  • Відселення останніх використаних ключів (LRU eviction)
  • Автоматичне відновлення після збою

Ви можете використовувати Redis з більшості існуючих мов програмування.

Redis написаний на ANSI C та робить на більшості POSIX системах, таких як Linux, *BSD, OS X без зовнішніх залежностей. Linux та OS X є двома операційними системами, де Redis був розроблений, та найбільше протестований, та ми рекомендуємо використовувати Linux для розгортання. Redis може робити на похідних від Solaris системах, таких як SmartOS, але підтримка є найкраща з можливих. Немає офіційної підтримки для побудов під Windows, але Microsoft розробляє та підтримує порт Rediss Win-64 .
