Починаємо роботу з sbt 

sbt використовує мале число концепцій, щоб підтримувати гнучкі та потужні визначення побудови. Є небагато концепцій, але sbt не є точно такою, як більшість інших систем побудови, та є деталі, де ви будете затинатись, якщо не прочитаєте цю документацію.

Керівництво Починаємо роботу охоплює концепції, що знадобляться вам, щоб створити та підтримувати визначення побудови sbt.

Є вкрай рекомендованим прочитати Починаємо роботу!

Якщо ви дуже поспішаєте, найбільш важливе концептууальне підгрунтя може бути знайдене в  .sbt визначення побудови, поле зору, та  більше типів налаштувань. Але ми не обіцяємо, що є гарною ідеєю пропустити інші сторінки цієї настанови.

Кращи читати за порядком, тому що пізніші сторінки Починаючи роботу будуються на концепціях того, що викладене раніше. 

Дякуємо що спробували sbt та насолоджуйтесь!


Встановлення sbt 

Щоб створити проект sbt вам треба виконати наступні кроки:

  • Встановити sbt та створити скрипт, щоб запустити його.

  • Встановити простий проект hello world.

    • Створити директорію проекту з первинними файлами в ньому.
    • Створити ваше визнаення побудови. 
  • Перейти до виконання, щоб вивчити, як виконувати sbt.
  • Потім перейти до .sbt визначення побудови, щоб визначити більше щодо визначень побудови.

Кінець кінцем, інсталяція sbt зводиться до стартового JAR та скрипта оболонки, але в залежності від платформи, ми провадимо декілька шляхів, щоб зробить цей процес меньш нудним. Переходьте до кроків для Mac, Windows, Linux, Lightbend Activator, або ручної інсталяції.

Підказки та зауваження

Якщо ви маєте жодні проблеми виконання sbt, дивіться Зауваження щодо встановлення щодо кодування терміналу, HTTP проксі, та опцій JVM.

Встановлення з універсального пакунку

Зауважте пакунок в вигляді ZIP або TGZ, та розкрийте його.

Встановлення на Mac з пакунку третіх сторін

Зауваження: Пакунки третіх сторін можуть не провадити останню версію. Будь ласка, переконайтесь, що повідомили про любі проблеми з ціма пакунками до їх розробників. 

Macports 

$ port install sbt

Homebrew 

$ brew install sbt

Встановлення на Windows

Завантажте msi інсталятор, та встановіть його.

Встановлення на Linux

DEB пакунок офіційно підтримується sbt.

Ubuntu та інші похідні від Debian дистрибутиви використовують формат DEB, але звичайно ви не інсталюєте ваш софт з локального файла DEB. Замість вони роблять з менедерами пакунків, або командного рядка  (як  apt-get, aptitude) або з графічним інтерфейсом (Synaptic). Виконайте наступні команди в консолі, щоб встановити sbt (вам знадобляться права адміна).

echo "deb https://dl.bintray.com/sbt/debian /" | sudo tee -a /etc/apt/sources.list.d/sbt.list
sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 642AC823
sudo apt-get update
sudo apt-get install sbt

Менеджери пакунків будуть перевіряти декілька репозитаріїв для пошуку пакунків. Бінарії sbt публікуються в Bintray, так що Bintray провадить APT репозитарій.

Hello, World 

This page assumes you’ve installed sbt.

Create a project directory with source code 

A valid sbt project can be a directory containing a single source file. Try creating a directory hello with a file hw.scala, containing the following:

object Hi {
  def main(args: Array[String]) = println("Hi!")
}

Now from inside the hello directory, start sbt and type run at the sbt interactive console. On Linux or OS X the commands might look like this:

$ mkdir hello
$ cd hello
$ echo 'object Hi { def main(args: Array[String]) = println("Hi!") }' > hw.scala
$ sbt
...
> run
...
Hi!

In this case, sbt works purely by convention. sbt will find the following automatically:

  • Sources in the base directory
  • Sources in src/main/scala or src/main/java
  • Tests in src/test/scala or src/test/java
  • Data files in src/main/resources or src/test/resources
  • jars in lib

By default, sbt will build projects with the same version of Scala used to run sbt itself.

You can run the project with sbt run or enter the Scala REPL with sbt console. sbt consolesets up your project’s classpath so you can try out live Scala examples based on your project’s code.

Build definition 

Most projects will need some manual setup. Basic build settings go in a file called build.sbt, located in the project’s base directory.

For example, if your project is in the directory hello, in hello/build.sbt you might write:

lazy val root = (project in file(".")).
  settings(
    name := "hello",
    version := "1.0",
    scalaVersion := "2.11.7"
  )

In .sbt build definition you’ll learn more about how to write a build.sbt file.

If you plan to package your project in a jar, you will want to set at least the name and version in abuild.sbt.

Setting the sbt version 

You can force a particular version of sbt by creating a file hello/project/build.properties. In this file, write:

sbt.version=0.13.11

to force the use of sbt 0.13.11. sbt is 99% source compatible from release to release. Still, setting the sbt version in project/build.properties avoids any potential confusion.
