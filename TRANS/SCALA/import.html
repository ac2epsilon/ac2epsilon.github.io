Scala import

Імпортування в Scala - це механізм, що дозволяє більш прямі посилання на різні сутності, такі як пакунки, класи, об'єкти, екземпляри, поля або методи. Концепція імпортування коду більш гнучка в Scala, ніж в Java або C++.

  • Твердження import меоже розташовуватись будь-де. Воно може бути на початку класу, в самому класі або об'єкті, або а методі або блоці.
  • Члени можуть бути переіменовані або приховані під час імпортування.
  • Пакунки або об'єкти можуть бути імпортовані в поточне поле зору.

Import

Кожний імпорт визначається ключовим словом "import". Простий приклад імпортування об'єкта, що міститься в пакунку:

package p1 {
  class A
}
object B {
  val x = new p1.A
  import p1.A
  val y = new A
}

В рядках 1-3 декларується пакунок "p1", що містить клас"A". В рядках 4-8 визначаєтсья об'єкт "B". В рядку 5 значенню "x" присвобється новий екземпляр класу "A" з використанням повного ідентифікатора. В рядку 6 зустрічається ключове слово та об'єкт імпорту "import p1.A". З цього місця на клас "A" можна посилатись за скороченим ідентифікатором, як це робиться в рядку 7.

можна імпортувати такі сутності, як поля або методи об'єктів:

import math.Pi
import math.round

println("Pi is: " + Pi) //Prints "Pi is: 3.141592653589793".
println("Pi rounded is: " + round(Pi)) //Prints "Pi rounded is: 3".

В першому рядку інпортуєтсья поле "Pi" об'єкта "math", в другому рядку метод "round" з об'єкту "math". Далі показані приклади використання ціх об'єктів без вживання повних ідентифікаторів "math.Pi" та "math.round".

Члени екземплярів також можуть бути імпортовані:

class Cube(val a:Double)
def volume(cube:Cube) = {
  import cube.a
  a*a*a
}
val cube1 = new Cube(4)
println("The volume of cube1 is: " + volume(cube1)) //Prints "The volume of cube1 is: 64.0".

В першому рядку декларуєтся клас "Cube" зі значенням "a". В рядках  2-5 визначаєтсья метод обчислення об'єму. В рядку 3 імопортується поле "a"аргументу "cube", та в наступному рядку іде еосилання на нього. Новий куб створюєтсья в рядку 6, та в рядку 7 друкуєтсья його о'бєм.

Пакунки також можуть бути імпортовані:

package p1.p2 {
  class A
}

import p1.p2

val a = new p2.A

В перших рядках декларуються два вкладені пакунки та клас "A". Далі імпортуєтсья "p2". Після цього посилання на внутрішній пакунок можливе без посилання на зовнішній "p1.p2".

Ви можете імпортувати класи з пакунку настунпим чином.

 import math.{Pi,round}

Класи math.Pi та math.round імпортуються в одному рядку.

Узагальнення

Символ _ використовується в Scala як * в Java. В прикладі нижче все імпортується з пакунку math.

 import math._
 println("Pi is: " + Pi) //Prints "Pi is: 3.141592653589793".
 println("Pi rounded is: " + round(Pi)) //Prints "Pi rounded is: 3".

Import selector clause

Renaming

Щоб запобігти колізіям ви можете переіменувати члени при імпорті. In the below example Pi is renamed as myPi and round as myRound while importing from the math package so that it doesn't collide with the local Pi and round members of the object.

import math.{Pi => myPi,round => myRound}
object myMaths extends  App {
  val Pi = 3.1 
  val round = TRUE
  println("Pi is: " + myPi) //Prints "Pi is: 3.141592653589793".
  println("Pi rounded is: " + myRound(myPi)) //Prints "Pi rounded is: 3".
}

Приховування

Scala надає можливість приховати один або більше класів, та імпортувати інші з пакунку.

import math.{Pi => _, _}
object myMaths extends  App {
  println("Pi is: " + Pi) // Error, Pi is hidden
}

В In the above example Pi is hidden and every thing else from the math package is imported into the scope. Accessing Pi in the example would give as an error as the Pi is hidden while importing.

Обмеження поля перегляду

Ви можете використовувати import будь-де, і це дозволяє котролювати поле перегляду імпортованих членів.

Ви можете використовувати import на вищому рівні пакунку.

package importTesting
import math.Pi
class test1 {
	def printPi {
		println("Pi is: " + Pi) //Successful
	}
}

class test2 {
	def printMyPi {
		println("Pi is :" + Pi) //Successful
	}
}

В прикладі вище math.pi імпортовано на вищому рівні пакунку, тому його видімість веь пакунок. Його можна використовувати в кожному класі пакунку.

Ви можете імпортувати для окремого класу.

package importTesting
class test1 {
	import math.Pi
	def printPi {
		println("Pi is: " + Pi) //Successfull
	}
}

class test2 {
	def printMyPi {
		println("Pi is :" + Pi) //Fail as the scope of Pi is limited to class test1
	}
}

В цьому прикладі ми імопртуємо в клас test1, та поле видимості Pi обмежено класом test1, та не досяжне у класі test2.

Ви можете імпортувати в методі та обмежети дію цім членом.

package importTesting
class test1 {
	def printPi {
		import math.Pi
		println("Pi is: " + Pi) //Successfull
	}

	def printMyPi {
		println("Pi is :" + Pi) //Fail as the scope of Pi is limited to the method
	}
}
